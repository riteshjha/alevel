1 Object-Oriented Programming (OOP)

Object-Oriented Programming (OOP) models a program as a collection of interacting objects, each combining data (attributes) and behaviour (methods). It promotes code reuse, modularity, and maintainability.

Class

A blueprint or template that defines the attributes and methods shared by all objects of that type. A class is not an object itself — it is the definition.

Object

An instance of a class. Each object has its own copy of the attributes but shares the class's method definitions. Creating an object is called instantiation.

Attribute

A variable that belongs to a class or object. Stores the state (data) of an object. Also called an instance variable or field.

Method

A function defined inside a class that describes the behaviour of objects. Methods can read or modify attributes.

Constructor

A special method called automatically when an object is created. Initialises the object's attributes. In Python: __init__. In pseudocode: NEW ClassName().

Encapsulation

Hiding internal data by making attributes private. External code accesses them only through getter and setter methods, protecting data integrity.

Class Definition — Pseudocode

Cambridge Pseudocode CLASS BankAccount PRIVATE AccountNumber : STRING PRIVATE Balance : REAL PRIVATE Owner : STRING PUBLIC PROCEDURE NEW(AccNum : STRING, InitBalance : REAL, OwnerName : STRING) AccountNumber ← AccNum Balance ← InitBalance Owner ← OwnerName ENDPROCEDURE PUBLIC FUNCTION GetBalance() RETURNS REAL // getter RETURN Balance ENDFUNCTION PUBLIC FUNCTION GetOwner() RETURNS STRING RETURN Owner ENDFUNCTION PUBLIC PROCEDURE Deposit(Amount : REAL) IF Amount > 0 THEN Balance ← Balance + Amount ENDIF ENDPROCEDURE PUBLIC FUNCTION Withdraw(Amount : REAL) RETURNS BOOLEAN IF Amount > 0 AND Amount <= Balance THEN Balance ← Balance - Amount RETURN TRUE ELSE RETURN FALSE ENDIF ENDFUNCTION ENDCLASS // Instantiation and use DECLARE MyAccount : BankAccount MyAccount ← NEW BankAccount("ACC001", 500.00, "Alice") CALL MyAccount.Deposit(200.00) OUTPUT MyAccount.GetBalance() // outputs 700.00

Class Definition — Python

Python class BankAccount: def __init__(self, acc_num, balance, owner): self.__account_number = acc_num # private (name-mangled) self.__balance = balance self.__owner = owner def get_balance(self): # getter return self.__balance def get_owner(self): return self.__owner def deposit(self, amount): if amount > 0: self.__balance += amount def withdraw(self, amount): if 0 < amount <= self.__balance: self.__balance -= amount return True return False # Instantiation and use my_account = BankAccount("ACC001", 500.00, "Alice") my_account.deposit(200.00) print(my_account.get_balance()) # 700.0
Exam tip: Cambridge expects you to know the difference between PUBLIC and PRIVATE. Attributes should always be PRIVATE and accessed only through PUBLIC methods. If asked to "write a class," always include a constructor (NEW), getters for any private attributes needed externally, and appropriate methods.

2 Inheritance & Polymorphism

Inheritance allows a subclass (child class) to acquire all the attributes and methods of a superclass (parent class). The subclass can add new attributes/methods and override inherited ones.

Inheritance Hierarchy

Shape (superclass)
↓ inherits
Circle (subclass)
Rectangle (subclass)
Triangle (subclass)

Each subclass inherits Shape's attributes (e.g. Colour) and methods (e.g. GetColour) and adds its own (e.g. Radius for Circle).

Inheritance — Pseudocode

Cambridge Pseudocode CLASS Shape PRIVATE Colour : STRING PUBLIC PROCEDURE NEW(C : STRING) Colour ← C ENDPROCEDURE PUBLIC FUNCTION GetColour() RETURNS STRING RETURN Colour ENDFUNCTION PUBLIC FUNCTION Area() RETURNS REAL RETURN 0 // overridden in subclasses ENDFUNCTION ENDCLASS // Circle INHERITS FROM Shape CLASS Circle INHERITS Shape PRIVATE Radius : REAL PUBLIC PROCEDURE NEW(C : STRING, R : REAL) SUPER.NEW(C) // call parent constructor Radius ← R ENDPROCEDURE PUBLIC FUNCTION Area() RETURNS REAL // overrides Shape.Area() RETURN 3.14159 * Radius * Radius ENDFUNCTION PUBLIC FUNCTION GetRadius() RETURNS REAL RETURN Radius ENDFUNCTION ENDCLASS // Instantiation DECLARE C1 : Circle C1 ← NEW Circle("Red", 5.0) OUTPUT C1.GetColour() // "Red" — inherited from Shape OUTPUT C1.Area() // 78.53975 — overridden in Circle

Polymorphism

Polymorphism means "many forms." The same method name (e.g. Area()) produces different behaviour depending on which class the object belongs to. This is achieved through method overriding.

Polymorphism in action
1
Both Circle and Rectangle inherit from Shape and both override Area().
2
If you call S.Area() where S is a Circle, Python/the runtime calls Circle's Area method.
3
If S is a Rectangle, the same call S.Area() calls Rectangle's Area method.
4
The calling code does not need to know which type of Shape it holds — the correct method is selected automatically at runtime.

Inheritance — Python

Python class Shape: def __init__(self, colour): self.__colour = colour def get_colour(self): return self.__colour def area(self): return 0 class Circle(Shape): # inherits from Shape def __init__(self, colour, radius): super().__init__(colour) # call parent constructor self.__radius = radius def area(self): # overrides Shape.area() return 3.14159 * self.__radius ** 2 class Rectangle(Shape): def __init__(self, colour, width, height): super().__init__(colour) self.__width = width self.__height = height def area(self): # overrides Shape.area() return self.__width * self.__height # Polymorphism — same method call, different behaviour shapes = [Circle("Red", 5), Rectangle("Blue", 4, 6)] for s in shapes: print(s.area()) # 78.53... then 24 — different results, same call
ConceptDefinitionKey benefit
InheritanceSubclass gains all attributes and methods of superclassCode reuse — common behaviour written once in superclass
Method OverridingSubclass provides its own version of an inherited methodSpecialisation — each subclass behaves appropriately
PolymorphismSame method name works differently per classFlexible code — one interface, many implementations
super()Calls a method from the parent classExtend parent behaviour rather than replace it entirely

3 Recursion

A recursive function is one that calls itself as part of its own definition. Every recursive solution must have:

  • Base case — a condition that stops the recursion and returns a value directly
  • Recursive case — reduces the problem towards the base case and calls the function again
Infinite recursion: If there is no base case, or the recursive case never reaches it, the function calls itself forever until the call stack overflows — causing a stack overflow error.

Factorial — Pseudocode & Trace

Cambridge Pseudocode FUNCTION Factorial(n : INTEGER) RETURNS INTEGER IF n <= 1 THEN RETURN 1 // base case ELSE RETURN n * Factorial(n - 1) // recursive case ENDIF ENDFUNCTION
Trace: Factorial(4)
1
Factorial(4) → 4 ≠ 1, calls Factorial(3)
2
Factorial(3) → 3 ≠ 1, calls Factorial(2)
3
Factorial(2) → 2 ≠ 1, calls Factorial(1)
4
Factorial(1) → base case → returns 1
5
Factorial(2) resumes → returns 2 × 1 = 2
6
Factorial(3) resumes → returns 3 × 2 = 6
7
Factorial(4) resumes → returns 4 × 6 = 24

The Call Stack

Each function call pushes a stack frame onto the call stack. The frame stores the local variables and the return address. When the function returns, its frame is popped.

Calling phase (building up)

Factorial(4) — waiting
Factorial(3) — waiting
Factorial(2) — waiting
Factorial(1) — returns 1

Returning phase (unwinding)

Factorial(4) → returns 24
Factorial(3) → returns 6
Factorial(2) → returns 2

Fibonacci — Recursive

Cambridge Pseudocode // Fibonacci: F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2) FUNCTION Fibonacci(n : INTEGER) RETURNS INTEGER IF n = 0 THEN RETURN 0 // base case 1 ELSE IF n = 1 THEN RETURN 1 // base case 2 ELSE RETURN Fibonacci(n-1) + Fibonacci(n-2) // recursive case ENDIF ENDIF ENDFUNCTION

Binary Search — Recursive Version

Cambridge Pseudocode FUNCTION BinSearch(Data : ARRAY[1:100] OF INTEGER, Low : INTEGER, High : INTEGER, Target : INTEGER) RETURNS INTEGER IF Low > High THEN RETURN -1 // base case — not found ELSE Mid ← (Low + High) DIV 2 IF Data[Mid] = Target THEN RETURN Mid // base case — found ELSE IF Target < Data[Mid] THEN RETURN BinSearch(Data, Low, Mid-1, Target) // search left ELSE RETURN BinSearch(Data, Mid+1, High, Target) // search right ENDIF ENDIF ENDIF ENDFUNCTION

Recursion vs Iteration

FeatureRecursionIteration
Code clarityOften cleaner for naturally recursive problems (trees, fractals)Clearer for simple repeated tasks
Memory useEach call uses stack space — risk of stack overflow on deep recursionUses constant extra memory
SpeedFunction call overhead per stepGenerally faster
Use whenProblem is naturally self-similar (tree traversal, divide-and-conquer)Simple loops over a sequence
Exam tip: Cambridge questions on recursion often ask you to (1) identify the base case and recursive case, (2) trace the calls for a small input, or (3) rewrite a recursive function iteratively. Always state both cases explicitly — one mark each is common.

4 File Handling

Files provide persistent storage — data survives after the program ends. Cambridge 9618 requires you to handle text files using pseudocode file operations.

File Modes

ModeKeywordBehaviour
ReadFOR READOpens an existing file for reading. File pointer starts at the beginning.
WriteFOR WRITECreates a new file (or overwrites an existing one). Data is written from the start.
AppendFOR APPENDOpens an existing file and adds new data at the end. Existing content is preserved.

Reading a File

Cambridge Pseudocode DECLARE Line : STRING DECLARE Total : INTEGER DECLARE Count : INTEGER Total ← 0 Count ← 0 OPENFILE "scores.txt" FOR READ WHILE NOT EOF("scores.txt") DO READFILE "scores.txt", Line Total ← Total + STR_TO_NUM(Line) Count ← Count + 1 ENDWHILE CLOSEFILE "scores.txt" IF Count > 0 THEN OUTPUT "Average: ", Total / Count ENDIF

Writing a File

Cambridge Pseudocode // FOR WRITE — creates/overwrites the file OPENFILE "results.txt" FOR WRITE FOR i ← 1 TO 5 INPUT Score WRITEFILE "results.txt", NUM_TO_STR(Score) NEXT i CLOSEFILE "results.txt" // FOR APPEND — adds to existing file OPENFILE "log.txt" FOR APPEND WRITEFILE "log.txt", "New entry added" CLOSEFILE "log.txt"

File Handling — Python Equivalent

Python # Reading with open("scores.txt", "r") as f: for line in f: print(line.strip()) # Writing (overwrites) with open("results.txt", "w") as f: for score in [85, 72, 91]: f.write(str(score) + "\n") # Appending with open("log.txt", "a") as f: f.write("New entry\n")
Always close files! In pseudocode you must explicitly write CLOSEFILE. Forgetting to close a file can cause data loss (buffered writes not flushed) or lock the file from other processes. In Python, the with statement closes the file automatically.
Exam tip: Cambridge mark schemes check for: correct OPENFILE with mode, READFILE/WRITEFILE with the filename repeated, EOF check in a WHILE loop for reading, and CLOSEFILE. Each of these is typically worth a separate mark.

5 Exception Handling

An exception is a runtime error that interrupts normal program flow (e.g. dividing by zero, reading a non-existent file, converting a non-numeric string). Exception handling lets a program catch these errors and respond gracefully rather than crashing.

Common Exception Types

ExceptionCauseExample
ZeroDivisionErrorDividing by zero10 / 0
ValueErrorWrong value type (e.g. converting non-number to int)int("hello")
FileNotFoundErrorOpening a file that does not existopen("missing.txt")
IndexErrorAccessing an index outside the array/list boundsarr[100] on a list of 5
TypeErrorOperation on incompatible types"5" + 3

TRY … EXCEPT — Pseudocode

Cambridge Pseudocode TRY INPUT NumStr Num ← STR_TO_NUM(NumStr) OUTPUT "Double is: ", Num * 2 EXCEPT OUTPUT "Error: please enter a valid number" ENDTRY

try / except / else / finally — Python

Python try: num_str = input("Enter a number: ") num = int(num_str) result = 100 / num except ValueError: print("Not a valid integer.") except ZeroDivisionError: print("Cannot divide by zero.") else: # runs only if no exception was raised print(f"Result: {result}") finally: # always runs — e.g. close resources print("Done.")

Exception Handling with Files — Python

Python try: with open("data.txt", "r") as f: content = f.read() print(content) except FileNotFoundError: print("Error: file not found.") except PermissionError: print("Error: no permission to read this file.")
BlockWhen it runsPurpose
tryAlways — contains the risky codeCode that might raise an exception
exceptOnly if a matching exception is raisedHandle the error gracefully
elseOnly if no exception was raisedCode that should run on success
finallyAlways, whether or not an exception occurredClean-up: close files, release resources

6 Scope & Parameter Passing

Understanding where variables exist and how values are passed between routines is essential for writing correct modular programs.

Variable Scope

ScopeDefinitionLifetimeAccess
Local variableDeclared inside a procedure or functionCreated when routine starts; destroyed when it endsOnly visible within that routine
Global variableDeclared outside all procedures/functions, at the top levelExists for the entire duration of the programVisible to all routines
Avoid global variables where possible. If multiple routines modify a global variable, bugs become hard to trace. Prefer passing data as parameters and returning results — this makes each routine self-contained and testable.
Cambridge Pseudocode DECLARE GlobalCount : INTEGER // global — visible everywhere GlobalCount ← 0 PROCEDURE Example() DECLARE LocalVal : INTEGER // local — only visible inside Example LocalVal ← 10 GlobalCount ← GlobalCount + 1 OUTPUT LocalVal, GlobalCount ENDPROCEDURE // LocalVal cannot be accessed here — it is out of scope

Parameter Passing: BYVAL vs BYREF

MethodHow it worksEffect on originalUse when
BYVAL (by value)A copy of the variable's value is passed. The routine works on the copy.Original variable is unchangedThe routine only needs to read the value; caller's variable should be protected
BYREF (by reference)The memory address of the variable is passed. The routine works on the original.Any changes inside the routine modify the originalThe routine needs to update the caller's variable (e.g. Swap, Sort)
Cambridge Pseudocode // BYVAL — original is unchanged PROCEDURE DoubleByVal(BYVAL X : INTEGER) X ← X * 2 OUTPUT X // outputs doubled value ENDPROCEDURE DECLARE Num : INTEGER Num ← 5 CALL DoubleByVal(Num) OUTPUT Num // still 5 — original unchanged // BYREF — original IS changed PROCEDURE DoubleByRef(BYREF X : INTEGER) X ← X * 2 ENDPROCEDURE Num ← 5 CALL DoubleByRef(Num) OUTPUT Num // now 10 — original was modified // Classic BYREF use case: Swap PROCEDURE Swap(BYREF A : INTEGER, BYREF B : INTEGER) DECLARE Temp : INTEGER Temp ← A A ← B B ← Temp ENDPROCEDURE
Exam tip: Cambridge questions often ask you to state whether a parameter should be passed BYVAL or BYREF and justify why. The key question is: does the routine need to change the caller's variable? If yes → BYREF. If no → BYVAL (safer, protects the original).

Practice Question 1 — 6 marks

A program models library books. Write a class Book with private attributes Title (STRING), Author (STRING), and Available (BOOLEAN). Include a constructor, a getter for each attribute, and a method Borrow() that sets Available to FALSE if the book is currently available and returns TRUE, or returns FALSE if it is already borrowed.

Cambridge Pseudocode CLASS Book PRIVATE Title : STRING PRIVATE Author : STRING PRIVATE Available : BOOLEAN PUBLIC PROCEDURE NEW(T : STRING, A : STRING) Title ← T Author ← A Available ← TRUE ENDPROCEDURE PUBLIC FUNCTION GetTitle() RETURNS STRING RETURN Title ENDFUNCTION PUBLIC FUNCTION GetAuthor() RETURNS STRING RETURN Author ENDFUNCTION PUBLIC FUNCTION IsAvailable() RETURNS BOOLEAN RETURN Available ENDFUNCTION PUBLIC FUNCTION Borrow() RETURNS BOOLEAN IF Available THEN Available ← FALSE RETURN TRUE ELSE RETURN FALSE ENDIF ENDFUNCTION ENDCLASS
Practice Question 2 — 5 marks

Write a recursive function SumArray that takes an array of integers and an index n, and returns the sum of elements from index 1 to n. Identify the base case and recursive case, and trace the calls for the array [3, 7, 2, 5] with n = 4.

Cambridge Pseudocode FUNCTION SumArray(Data : ARRAY[1:100] OF INTEGER, n : INTEGER) RETURNS INTEGER IF n = 0 THEN RETURN 0 // base case ELSE RETURN Data[n] + SumArray(Data, n-1) // recursive case ENDIF ENDFUNCTION

Base case: n = 0 → return 0 (empty sum).
Recursive case: return Data[n] + SumArray(Data, n-1).

Trace for [3,7,2,5], n=4:
SumArray(4) = 5 + SumArray(3)
SumArray(3) = 2 + SumArray(2)
SumArray(2) = 7 + SumArray(1)
SumArray(1) = 3 + SumArray(0)
SumArray(0) = 0 (base case)
→ 3+0=3 → 7+3=10 → 2+10=12 → 5+12 = 17

Practice Question 3 — 4 marks

A program reads student names from a file "students.txt" and writes those whose names start with the letter 'A' to a new file "anames.txt". Write the pseudocode for this task.

Cambridge Pseudocode DECLARE Name : STRING OPENFILE "students.txt" FOR READ OPENFILE "anames.txt" FOR WRITE WHILE NOT EOF("students.txt") DO READFILE "students.txt", Name IF UCASE(SUBSTRING(Name, 1, 1)) = 'A' THEN WRITEFILE "anames.txt", Name ENDIF ENDWHILE CLOSEFILE "students.txt" CLOSEFILE "anames.txt"
Practice Question 4 — 4 marks

The following procedure has a bug related to parameter passing. Identify the bug, explain why it occurs, and state how to fix it.

Cambridge Pseudocode PROCEDURE Swap(BYVAL A : INTEGER, BYVAL B : INTEGER) DECLARE Temp : INTEGER Temp ← A A ← B B ← Temp ENDPROCEDURE

Bug: The parameters are declared BYVAL instead of BYREF.

Why it occurs: BYVAL passes copies of A and B into the procedure. The swap operates on those copies — when the procedure ends, the copies are discarded and the caller's original variables remain unchanged. The swap has no effect outside the procedure.

Fix: Change both parameters to BYREF:

Cambridge Pseudocode PROCEDURE Swap(BYREF A : INTEGER, BYREF B : INTEGER)

Now A and B refer to the caller's original variables. Changes inside the procedure modify the originals directly.

Practice Question 5 — 5 marks

A subclass SavingsAccount inherits from BankAccount (from Section 1). It adds a REAL attribute InterestRate and a method AddInterest() that increases the balance by InterestRate percent. Write the class definition in pseudocode.

Cambridge Pseudocode CLASS SavingsAccount INHERITS BankAccount PRIVATE InterestRate : REAL PUBLIC PROCEDURE NEW(AccNum : STRING, InitBalance : REAL, OwnerName : STRING, Rate : REAL) SUPER.NEW(AccNum, InitBalance, OwnerName) InterestRate ← Rate ENDPROCEDURE PUBLIC FUNCTION GetInterestRate() RETURNS REAL RETURN InterestRate ENDFUNCTION PUBLIC PROCEDURE AddInterest() DECLARE Interest : REAL Interest ← GetBalance() * InterestRate / 100 CALL Deposit(Interest) ENDPROCEDURE ENDCLASS

Note: AddInterest uses the inherited GetBalance() and Deposit() methods — it cannot directly access the private Balance attribute of BankAccount.

OOP Key Terms

  • Class — blueprint/template
  • Object — instance of a class
  • Encapsulation — private data + public methods
  • Inheritance — subclass gets superclass members
  • Polymorphism — same name, different behaviour
  • Constructor — NEW / __init__

Recursion Rules

  • Must have a base case
  • Recursive case must move toward base
  • Each call uses a stack frame
  • No base case → stack overflow

File Pseudocode

  • OPENFILE "f" FOR READ
  • OPENFILE "f" FOR WRITE
  • OPENFILE "f" FOR APPEND
  • READFILE "f", Var
  • WRITEFILE "f", Var
  • EOF("f") → BOOLEAN
  • CLOSEFILE "f"

Common Mistakes

  • Using BYVAL when the routine must change the caller's variable
  • Accessing a superclass's PRIVATE attribute directly from a subclass — use inherited getters
  • Forgetting CLOSEFILE — always close after use
  • Missing base case in recursion — causes infinite recursion
  • Forgetting to call SUPER.NEW() in a subclass constructor
  • Using FOR WRITE on a file you want to append — it overwrites everything
