1 Data Transmission

When data moves between devices it must travel as a physical signal — an electrical voltage change in a wire, a pulse of light in fibre, or a radio wave. Understanding how bits are sent reliably is the foundation of networking.

1.1 Serial vs Parallel Transmission

🔗 Serial Transmission

  • Bits sent one at a time down a single wire
  • Slower throughput for the same clock speed
  • No timing skew between bits
  • Reliable over long distances
  • Lower cost — fewer wires
  • Examples: USB, SATA, Ethernet, HDMI

🔀 Parallel Transmission

  • Multiple bits sent simultaneously on multiple wires
  • Higher throughput at same clock speed
  • Skew problem: bits arrive at slightly different times
  • Only reliable over short distances
  • Higher cost — many wires required
  • Examples: Internal CPU buses, old printer ports
ℹ️ Why has serial overtaken parallel? Modern serial buses (USB 3, PCIe, SATA) run at such high clock speeds that they out-perform parallel buses despite sending one bit at a time. Skew limits parallel buses to short distances; serial has no such problem.

1.3 Key Transmission Terms

Bit Rate
The number of bits transmitted per second (bps). The actual data throughput. Higher = faster transfer.
Baud Rate
The number of signal changes (symbols) per second. One symbol can carry more than one bit if multiple signal levels are used.
Bandwidth
The maximum data transfer rate of a channel, measured in bps (or Mbps, Gbps). Determined by the physical medium.
Latency
The delay between a signal being sent and received. Measured in milliseconds. Affected by distance, medium, and intermediate devices.
Simplex
Data flows in one direction only. Example: TV broadcast, keyboard → computer.
Half-Duplex
Data flows in both directions but not simultaneously. One party must wait. Example: walkie-talkie, Wi-Fi.
Full-Duplex
Data flows in both directions simultaneously. Example: telephone call, most Ethernet connections.
Bit Rate vs Baud Rate
Bit Rate = Baud Rate × Bits per Symbol
If each symbol represents 4 signal levels (2 bits), and baud rate = 1000: bit rate = 1000 × 2 = 2000 bps
✎ Worked Example — Bit rate calculation
QA modem uses 8 distinct signal levels per symbol and a baud rate of 4800. What is the bit rate?
1Each symbol has 8 levels. Bits per symbol = log₂(8) = 3 bits per symbol
2Bit rate = 4800 × 3 = 14,400 bps
Bit rate = 14,400 bps (14.4 kbps)
✏ Practice Questions

(a) Explain the difference between bit rate and baud rate.
(b) A channel uses 16 distinct signal levels. If the baud rate is 2400, calculate the bit rate.
(c) State two reasons why serial transmission is preferred over parallel for long-distance communication.

(a) Bit rate is the number of bits transmitted per second; baud rate is the number of signal changes (symbols) per second. Bit rate ≥ baud rate when each symbol encodes multiple bits.

(b) log₂(16) = 4 bits per symbol. Bit rate = 2400 × 4 = 9600 bps.

(c) Any two: (1) No timing skew — all bits travel on one wire so arrive in order. (2) Cheaper — fewer wires needed. (3) Less interference/crosstalk between wires.

2 Error Detection

Noise and interference can corrupt bits during transmission. Error detection methods allow the receiver to recognise (but not always correct) when corruption has occurred.

2.1 Parity Checking

A parity bit is added to each byte so that the total number of 1s in the group is always even (even parity) or always odd (odd parity). Both sender and receiver agree in advance which convention to use.

Even parity example — each row must have an even number of 1s:

Data (7 bits)Count of 1sParity bit addedFull byte (8 bits)
10110104 (even)010110100
11010014 (even)011010010
01101104 (even)001101100
10011014 (even)010011010
1111010 ← error!5 (odd)⚠ Parity violation detected!
⚠️ Limitation: Parity can only detect an odd number of bit errors. If two bits flip, the parity count remains even and the error goes undetected. Parity also cannot correct errors — it only flags that something went wrong.

Two-dimensional (block) parity:

Parity bits are added for both each row and each column, creating a matrix. This can detect and locate single-bit errors (though still cannot correct them on its own).

✎ Worked Example — Determine the even parity bit for 1100101
1Count 1s in 1100101: there are 4 ones (even)
2For even parity we need an even total, so parity bit = 0
3Full byte transmitted: 11001010
Parity bit = 0  (total 1s = 4, which is even ✓)

2.2 Checksum

A checksum is a value computed from a block of data before transmission. The receiver recalculates the checksum from the received data and compares it to the transmitted checksum. If they differ, an error occurred.

✎ Worked Example — Simple checksum (mod 256)
1Data bytes to send: 72, 101, 108, 108, 111 (the word "Hello")
2Sum = 72 + 101 + 108 + 108 + 111 = 500
3Checksum = 500 mod 256 = 244
4Send: 72, 101, 108, 108, 111, 244
5Receiver recalculates: 72+101+108+108+111 = 500; 500 mod 256 = 244. Matches → no error detected.
Checksum = 244. If any byte was corrupted, the recalculated checksum would differ.

2.3 Echo Checking

The receiver echoes back every character it receives to the sender. The sender compares the echo with what it originally sent. If they match, the data was received correctly; if not, the sender retransmits.

  • Advantage: Simple to implement; works at the application level
  • Disadvantage: Doubles the bandwidth used; adds round-trip latency
✏ Practice Questions

(a) A system uses odd parity. What is the parity bit for the data 0110011?
(b) A byte 10110101 is received. The system uses even parity. State whether an error has been detected and explain why.
(c) State one advantage of checksum over parity checking.

(a) 0110011 has four 1s (even). For odd parity we need an odd total → parity bit = 1. Full byte: 01100111.

(b) Count 1s in 10110101: 1+1+1+1+1 = 5 ones (odd). Even parity requires an even count. → An error has been detected because the number of 1s is odd, violating even parity.

(c) Any one: Checksum can detect multiple-bit errors in a block of data, whereas parity can miss even numbers of errors. / Checksum is applied to an entire block of data, not just a single byte, making it more robust.

3 Network Hardware

Different devices play specific roles in building and running a network. You need to know what each device does and at which layer of the network model it operates.

🔌 NIC

Network Interface Card. Gives a device a unique MAC address. Converts data to/from electrical or optical signals. Operates at the Data Link layer.

🔁 Repeater

Amplifies/regenerates a signal to extend its range. Operates at the Physical layer. Does not inspect data — just boosts the signal.

🌿 Hub

Connects multiple devices on a LAN. Broadcasts every frame to all ports — creates collisions. Largely replaced by switches. Physical layer device.

🔀 Switch

Like a hub but forwards frames only to the destination MAC address. Maintains a MAC address table. Reduces collisions. Data Link layer.

🌐 Router

Connects different networks. Routes packets using IP addresses. Maintains a routing table. Operates at the Network layer.

🚪 Gateway

Connects networks using different protocols. Performs protocol translation. Operates at multiple layers. Example: connecting a LAN to the internet via an ISP.

📡 Access Point

Connects wireless devices to a wired LAN. Extends a wired network wirelessly using Wi-Fi (IEEE 802.11). Acts as a hub for wireless devices.

🔗 Bridge

Connects two LAN segments at the Data Link layer. Filters traffic between segments using MAC addresses. Reduces unnecessary traffic.

📝 Exam tip — Switch vs Hub vs Router: The most commonly tested distinction. Switch uses MAC addresses to send frames to the right port. Hub broadcasts to everyone. Router uses IP addresses to forward packets between networks. Remember: switch stays within one network; router connects different networks.

✏ Practice Questions

(a) Describe two differences between a hub and a switch.
(b) Explain why a router is needed when connecting a home network to the Internet.
(c) State two advantages of fibre optic cable over copper twisted pair.

(a) Any two: (1) A switch uses MAC addresses to forward frames to a specific port; a hub broadcasts to all ports. (2) Switches reduce network collisions; hubs cause collisions on the shared medium. (3) Switches are more secure as data only reaches the intended device; hubs send data to all devices.

(b) The home network uses private IP addresses; the Internet uses public IP addresses. A router connects these two different networks by routing packets between them using IP addresses, and performs NAT (Network Address Translation) to map private addresses to the ISP's public address.

(c) Any two: (1) Much faster data transfer speeds. (2) Immune to electromagnetic interference. (3) Data can travel much longer distances without signal degradation. (4) More secure — harder to tap without detection.

4 Network Topologies

A network topology describes the physical or logical arrangement of devices and connections in a network. The Cambridge syllabus requires you to know bus, star, ring, and mesh topologies.

Bus Topology
  • Cheap — one cable
  • Easy to extend
  • Backbone failure = full outage
  • Collisions on shared medium
  • Terminators required
  • Poor scalability
Star Topology
  • One device fails → others unaffected
  • Easy to add devices
  • Centralised management
  • Good performance
  • Central switch failure = full outage
  • More cable than bus
  • Switch is expensive
Ring Topology
  • Equal access via token passing
  • No collisions
  • Predictable performance
  • One device failure disrupts ring
  • Difficult to add devices
  • Slower than star as data passes through every node
Mesh Topology
  • Highly resilient — multiple paths
  • No single point of failure
  • Fault tolerant
  • Very expensive — many cables
  • Complex to manage
  • Partial mesh is a compromise
💡 Physical vs Logical topology: The physical topology describes how devices are physically connected with cables. The logical topology describes how data actually flows. A network can be physically wired as a star but behave logically as a bus (as in early Ethernet using hubs).
✏ Practice Questions

(a) A school has 30 computers in a classroom. Describe the star topology and explain why it is more suitable than a bus topology for this classroom network.
(b) State the topology used by the Internet and explain why it was chosen.
(c) Explain the role of terminators in a bus topology.

(a) In a star topology, all devices connect to a central switch/hub [1]. Star is more suitable because: if one device fails it does not affect others [1]; easier to add/remove computers without disrupting the network [1]; switch reduces collisions as data is only sent to the destination device [1].

(b) The Internet uses a mesh (partial mesh) topology [1]. It was chosen because it provides multiple redundant paths between nodes — if one link or router fails, data can be routed around the failure [1], giving high resilience and fault tolerance [1].

(c) Terminators absorb the signal at each end of the bus cable [1], preventing the electrical signal from reflecting back along the cable which would cause interference/corrupt data [1].

✏ Practice Questions

(a) A business has 50 employees who need to share files securely. Recommend either client–server or P2P and justify your answer.
(b) Explain why P2P file sharing can be difficult to shut down.

(a) Client–server [1]. Justification: Access to shared files can be controlled centrally through user authentication on the server [1]; data can be backed up centrally and consistently [1]; easier for an IT administrator to manage 50 users' permissions and audit file access [1].

(b) In P2P there is no central server to shut down [1]. Files are distributed across thousands or millions of nodes worldwide [1]. Removing some nodes does not remove the content — other nodes continue to share it [1].

6 The Internet & TCP/IP

6.1 The TCP/IP Four-Layer Model

The TCP/IP model is the practical framework that governs how data is transmitted across the Internet. It has four layers, each with a specific responsibility. As data travels down the layers from sender to receiver, each layer adds a header (and sometimes a trailer) — this is called encapsulation.

6.3 Packet Switching

Rather than dedicating a fixed circuit between sender and receiver (as in circuit switching), the Internet uses packet switching: data is divided into small packets, each routed independently across the network.

6.4 DNS — Domain Name System

Humans use domain names (e.g. cambridge.org); computers use IP addresses. DNS translates between them.

1
You type cambridge.org in your browser. Your computer checks its local DNS cache — if found, uses that IP immediately.
2
If not cached, your computer asks the Recursive Resolver (usually run by your ISP or Google/1.1.1.1).
3
The resolver asks a Root Name Server → directed to the .org TLD name server.
4
The TLD server directs to the Authoritative Name Server for cambridge.org.
5
The authoritative server returns the IP address (e.g. 131.111.150.25).
6
Your browser connects to the IP address. The result is cached locally for next time.
✏ Practice Questions

(a) State three advantages of packet switching over circuit switching for internet data transfer.
(b) Explain the role of the Transport layer in the TCP/IP model when downloading a large file.
(c) State why IPv6 was introduced and give one technical difference between IPv4 and IPv6.

(a) Any three: (1) Efficient use of bandwidth — multiple users share the same links. (2) Resilient — if a link fails, packets are rerouted automatically. (3) No dedicated connection needed before data can be sent. (4) Scales well — adding more routers increases capacity.

(b) TCP (Transport layer) divides the file into segments and adds sequence numbers to each [1]. It ensures reliable delivery — if a segment is lost or corrupted the receiver sends a negative acknowledgement (NACK) and TCP retransmits the missing segment [1]. At the receiving end, TCP uses the sequence numbers to reassemble the segments in the correct order [1].

(c) IPv6 was introduced because IPv4's ~4.3 billion addresses were nearly exhausted due to the rapid growth of internet-connected devices [1]. Technical difference: IPv4 uses 32-bit addresses (4 billion possible); IPv6 uses 128-bit addresses (~3.4×10³⁸ possible) [1].

Email Protocol Comparison: POP3 vs IMAP

📥 POP3 (Post Office Protocol v3)

  • Downloads emails to local device
  • Emails typically deleted from server after download
  • Works offline after download
  • Best for single-device access
  • Less server storage needed
  • Risk: if local device lost, emails gone

🔄 IMAP (Internet Message Access Protocol)

  • Emails remain on the server
  • Local client displays synced copy
  • Requires internet connection to read
  • Best for multiple-device access
  • More server storage needed
  • Safe: emails always on server
📝 Exam tip — SMTP vs POP3/IMAP: SMTP is always for sending; POP3/IMAP are for receiving. You need both when configuring an email client. A common exam question asks you to select the correct protocol for a scenario and justify your choice.
✎ Worked Example — What happens when you send an email
1You compose and send the email in your email client.
2SMTP is used to send the email from your client to your outgoing mail server (e.g. smtp.gmail.com).
3Your mail server uses DNS to look up the recipient's mail server's IP address (the MX record for their domain).
4Your server uses SMTP again to forward the email to the recipient's incoming mail server.
5The recipient's email client uses POP3 or IMAP to retrieve the email from the server.
Sending uses SMTP; receiving uses IMAP or POP3. DNS is used to find the recipient's server.
✏ Practice Questions

(a) A user wants to access their email from their laptop, phone, and tablet. Explain whether POP3 or IMAP is more appropriate.
(b) Explain why protocols are necessary in computer networking.
(c) State the difference between HTTP and HTTPS.

(a) IMAP is more appropriate [1]. With IMAP, emails are stored on the server and synchronised to all devices — reading an email on the phone marks it as read on the laptop too [1]. POP3 would download emails to one device and delete them from the server, meaning other devices could not access them [1].

(b) Protocols define agreed standards for formatting, transmitting, and receiving data [1], so that devices from different manufacturers can communicate reliably [1]. Without protocols, there would be no common language and devices would not be able to interpret each other's data [1].

(c) HTTP transmits data in plaintext — it is unencrypted and can be intercepted. HTTPS uses TLS encryption to protect data in transit [1], and uses a digital certificate to verify the server's identity [1].

8 Network Security

(In-depth security is covered in Topic 7. Here we focus on the network-level security components.)

8.1 Firewalls

A firewall monitors and controls incoming and outgoing network traffic based on a set of security rules. It creates a barrier between a trusted internal network and untrusted external networks.

Packet Filtering
Examines each packet's source IP, destination IP, port number, and protocol. Accepts or drops based on a rule table. Fast but cannot inspect payload content.
Stateful Inspection
Tracks the state of active connections. Only allows packets that are part of an established, legitimate session. More thorough than simple packet filtering.
Application-level (Proxy) Firewall
Inspects traffic at the Application layer. Understands specific protocols (HTTP, FTP). Can block specific content or sites. Slowest but most thorough.
Firewall Rules
Specify what is allowed/denied based on: source IP, destination IP, port, protocol, direction (inbound/outbound). Rules are evaluated top to bottom; first match wins.

8.2 Proxy Servers

A proxy server acts as an intermediary between clients and the internet. Requests from clients go to the proxy, which makes the request on their behalf.

Web Caching
Stores copies of frequently-requested web pages. Subsequent requests are served from cache — reduces bandwidth use and speeds up access.
Content Filtering
Blocks access to specific websites or content categories (e.g. social media, adult content). Used in schools and workplaces.
Anonymity
External servers see the proxy's IP address, not the client's. Hides internal network structure from the outside world.
Access Control
Can require authentication before granting internet access. Logs all user activity for auditing.
✏ Practice Questions

(a) Describe two functions of a firewall in a company network.
(b) A school uses a proxy server. Explain two benefits this provides.
(c) Describe the difference between a packet-filtering firewall and a stateful inspection firewall.

(a) Any two: (1) Monitors and filters incoming packets — blocks traffic from unauthorised IP addresses or ports. (2) Prevents outgoing connections to known malicious sites. (3) Logs network traffic for security auditing. (4) Creates a boundary between the internal trusted network and the external untrusted internet.

(b) Any two: (1) Content filtering — blocks inappropriate websites, so pupils cannot access unsuitable content [1]. (2) Web caching — stores copies of frequently visited pages, reducing bandwidth usage and speeding up access for all users [1]. (3) Hides internal IP addresses — external sites only see the proxy's IP [1].

(c) Packet filtering examines each packet in isolation — only looks at header information (IP, port, protocol) without tracking connections [1]. Stateful inspection maintains a table of active connections and only allows packets that belong to an established legitimate session [1] — it is more secure because it prevents packets that claim to be part of a connection that was never established.

Question 1 [7 marks]

(a) [1] State the difference between serial and parallel data transmission.

(b) [2] A transmission system uses 4 signal levels per symbol. The baud rate is 3000. Calculate the bit rate.

(c) [2] Data 1011010 is to be transmitted using even parity. State the parity bit and the complete 8-bit pattern transmitted. Explain how the receiver detects whether an error has occurred.

(d) [2] State two reasons why serial transmission is used for long-distance data communication rather than parallel transmission.

(a) Serial transmits one bit at a time on a single wire; parallel transmits multiple bits simultaneously on multiple wires. [1]

(b) 4 signal levels → log₂(4) = 2 bits per symbol [1]. Bit rate = 3000 × 2 = 6000 bps [1]

(c) 1011010 has 4 ones (even) → parity bit = 0. Full byte: 10110100 [1].
Receiver counts the 1s in the received byte. If the count is odd (not even), a parity violation has occurred and an error is flagged [1].

(d) Any two: (1) No skew problem — all bits on one wire arrive in sequence. (2) Cheaper — fewer wires needed over long distances. (3) Less crosstalk/interference between adjacent wires. [1 each]
Question 2 [8 marks]

A company is designing a network for a new office building with 80 computers across three floors.

(a) [3] Compare bus and star topologies. In your answer, discuss reliability, performance, and cost.

(b) [2] The company decides on a star topology. Identify the central device that should be used and explain why a switch is preferable to a hub.

(c) [2] The building requires a connection to the Internet. Name the device that connects the company's LAN to the Internet and explain its role.

(d) [1] State the protocol used for accessing web pages securely.

(a) Reliability: In a star, a single device failing does not affect others; in a bus, a break in the backbone affects all devices [1]. Performance: A star with a switch dedicates bandwidth per device, reducing collisions; a bus is a shared medium — more devices = more collisions and lower performance [1]. Cost: Bus uses less cable (cheaper) but requires terminators; star requires more cable and a central switch (more expensive) [1].

(b) Central device: Switch [1]. A switch reads the MAC address of each incoming frame and forwards it only to the destination device's port, reducing unnecessary traffic; a hub broadcasts to all ports causing collisions and wasting bandwidth [1].

(c) Device: Router [1]. The router connects the company's private LAN (using private IP addresses) to the Internet. It routes packets between the two networks using IP addresses and performs NAT to map private addresses to the company's public IP [1].

(d) HTTPS [1]
Question 3 [6 marks]

A user sends an email from their home computer to a colleague.

(a) [2] Identify the protocol used to send the email and the protocol the colleague would use to receive the email on multiple devices. Justify your choice of receiving protocol.

(b) [2] Explain the role of DNS in the process of sending this email.

(c) [2] The email is transmitted as a series of packets. Describe two pieces of information contained in each packet's header.

(a) Sending: SMTP [1]. Receiving: IMAP [1] — because IMAP stores emails on the server and synchronises them across multiple devices; POP3 would download and delete from server, meaning only one device could access the emails.

(b) When the email server needs to deliver the message to the recipient's server, it uses DNS to perform a lookup on the recipient's domain name [1] to find the IP address of their mail server (specifically the MX record) [1], so the email can be routed to the correct server.

(c) Any two from: (1) Source IP address — identifies the sending device. (2) Destination IP address — identifies the receiving device. (3) Sequence number — used to reassemble packets in the correct order. (4) Packet number / total packet count. (5) Checksum / error detection value. [1 each]

Common Mistakes

  • Confusing bit rate (bits/s) with baud rate (symbols/s)
  • Saying a switch "broadcasts" — it doesn't, a hub does
  • Confusing SMTP (send) with POP3/IMAP (receive)
  • Saying parity detects all errors — it misses even-numbered errors
  • Confusing router (connects networks, uses IP) with switch (connects devices on same network, uses MAC)
  • Saying DNS "converts" URLs — it translates domain names to IP addresses
