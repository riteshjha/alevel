Algorithm Design & Problem-Solving

Topic 10 · Paper 2 · Pseudocode, sorting, searching, and complexity

📋 Cambridge Pseudocode Conventions

Cambridge Pseudocode // Variables and assignment DECLARE Count : INTEGER Count ← 0 // Selection IF Score >= 50 THEN OUTPUT "Pass" ELSE OUTPUT "Fail" ENDIF // Loops FOR i ← 1 TO 10 Total ← Total + i NEXT i WHILE Count < 5 DO Count ← Count + 1 ENDWHILE REPEAT INPUT Value UNTIL Value > 0

↕️ Sorting Algorithms

Bubble Sort — O(n²)

  • Compare adjacent pairs, swap if out of order
  • Repeat passes until no swaps needed
  • Simple but inefficient for large data sets

Insertion Sort — O(n²)

  • Build sorted portion left to right
  • Insert each element into correct position
  • Efficient for nearly-sorted data

Merge Sort — O(n log n)

  • Divide array into halves recursively
  • Merge sorted halves back together
  • Consistent performance; uses extra memory

Quick Sort — O(n log n) avg

  • Choose a pivot; partition into smaller/larger
  • Recursively sort partitions
  • Fast in practice; worst case O(n²)

🔎 Searching Algorithms

Linear Search — O(n)
Check each element in order. Works on unsorted data. Simple but slow for large lists.
Binary Search — O(log n)
Requires sorted data. Check middle element; eliminate half the remaining elements each step. Very efficient for large lists.
Cambridge Pseudocode // Binary search PROCEDURE BinarySearch(Data : ARRAY[1:100] OF INTEGER, Target : INTEGER) Low ← 1 High ← 100 Found ← FALSE WHILE Low <= High AND NOT Found DO Mid ← (Low + High) DIV 2 IF Data[Mid] = Target THEN Found ← TRUE ELSE IF Data[Mid] < Target THEN Low ← Mid + 1 ELSE High ← Mid - 1 ENDIF ENDWHILE ENDPROCEDURE

📦 Abstract Data Types

Stack (LIFO)

  • Last In, First Out
  • Operations: Push, Pop, Peek, isEmpty
  • Used by: recursion, undo operations, expression evaluation
  • Pointer: Top of stack

Queue (FIFO)

  • First In, First Out
  • Operations: Enqueue, Dequeue, isEmpty
  • Circular queue avoids wasted space
  • Used by: print spooling, task scheduling

Linked List

  • Nodes: data + pointer to next node
  • Dynamic size; no contiguous memory needed
  • Insertion/deletion efficient; traversal slow
  • Doubly linked: pointers to both next and previous

Binary Tree

  • Each node has at most 2 children (left, right)
  • BST: left < root < right
  • Traversals: in-order, pre-order, post-order
  • Used for: sorted data, expression trees

Hash Table

  • Hash function maps key to array index
  • O(1) average lookup, insert, delete
  • Collision handling: chaining or open addressing
  • Used for: dictionaries, caches, sets

🌲 Tree Traversals

Cambridge Pseudocode // In-order traversal (produces sorted output for a BST) PROCEDURE InOrder(Node) IF Node <> NULL THEN InOrder(Node.Left) // visit left subtree OUTPUT Node.Data // visit node InOrder(Node.Right) // visit right subtree ENDIF ENDPROCEDURE

Programming Concepts

Topic 12 · Paper 2 · OOP, recursion, file handling, and exception handling

🏗️ Object-Oriented Programming (OOP)

Class
A blueprint/template defining attributes (data) and methods (behaviour) for objects.
Object
An instance of a class. Has its own attribute values but shares the class's methods.
Encapsulation
Bundling data and methods together; hiding internal implementation using private attributes accessed via getters/setters.
Inheritance
A subclass inherits attributes and methods from a superclass. Promotes code reuse. Subclass can override methods.
Polymorphism
The same method name behaves differently in different classes. Achieved via method overriding in subclasses.
Constructor
Special method called when an object is created. Initialises attributes. In Python: __init__.
Python class Animal: def __init__(self, name, sound): self.__name = name # private attribute self.__sound = sound def speak(self): return f"{self.__name} says {self.__sound}" def get_name(self): # getter return self.__name class Dog(Animal): # inherits from Animal def __init__(self, name): super().__init__(name, "Woof") def fetch(self): # added method return "Fetching the ball!"

🔁 Recursion

ℹ️ A recursive function calls itself. Every recursive function needs a base case (stopping condition) and a recursive case that moves towards the base case.
Cambridge Pseudocode FUNCTION Factorial(n : INTEGER) RETURNS INTEGER IF n <= 1 THEN RETURN 1 // base case ELSE RETURN n * Factorial(n - 1) // recursive case ENDIF ENDFUNCTION

🔄 Software Development Lifecycle (SDLC)

PhaseDescription
AnalysisDefine the problem; gather requirements (functional and non-functional). Produce a requirements specification.
DesignSystem architecture, data structures, algorithms, user interface design, data flow diagrams.
ImplementationWrite and test code modules. Follow coding standards.
TestingVerify and validate the system meets requirements. Use test plans.
EvaluationAssess how well the solution meets requirements. Identify improvements.
MaintenanceCorrective (fix bugs), adaptive (new requirements), perfective (improvements).

🧪 Testing

Normal Test Data
Valid data within expected range. System should process and accept it without errors.
Boundary Test Data
Values at the extreme edges of valid range (e.g. if valid is 1–100, test with 1, 100, 0, 101).
Erroneous Test Data
Invalid data the system should reject (wrong type, out of range, empty). Tests validation routines.
White-Box Testing
Tester has full knowledge of code internals. Tests all branches and paths through the code.
Black-Box Testing
Tester has no knowledge of internals. Tests based on inputs and expected outputs only.
Stub Testing
Placeholder module used when the real module is not yet complete. Allows testing of other modules in isolation.

Further Data Representation

Topic 14 · Paper 3 · Floating point numbers, normalisation, and precision

🔠 Floating Point Representation

ℹ️ Floating point stores numbers as: mantissa × 2exponent. The mantissa holds significant digits; the exponent scales the number. Similar to scientific notation.
Mantissa
The significant digits of the number. Stored in two's complement (for signed). Determines precision.
Exponent
Determines the magnitude/scale. Stored in two's complement. Larger exponent = larger number representable.
Normalisation
Ensures the mantissa uses all available bits for precision. For positive: leading bits are 0.1…; for negative: leading bits are 1.0…
Underflow
Number too close to zero to be represented — stored as 0. Exponent too small/negative.
Overflow
Number too large to be represented. Exponent too large.
Rounding Error
Most real numbers cannot be represented exactly in binary. The nearest representable value is used — introduces a small error.
📝 Trade-off: More mantissa bits → more precision. More exponent bits → wider range. Given a fixed word size, you must choose. Cambridge exams often ask you to convert a floating point binary pattern and explain the effects of changing the bit allocation.

Further Hardware

Topic 15 · Paper 3 · RISC vs CISC, pipelining, and parallel processing

⚖️ RISC vs CISC

FeatureRISCCISC
Instruction setSmall, simpleLarge, complex
Execution time1 clock cycle (mostly)Multiple cycles
RegistersMany general-purposeFewer, specialised
Memory accessLoad/store onlyOperations directly on memory
PipeliningEasierHarder (variable-length instructions)
ExamplesARM (mobile), RISC-Vx86 (Intel/AMD)

🚰 Pipelining

Pipeline
Overlapping stages of the FDE cycle for different instructions. While one instruction is being decoded, the next is being fetched.
Pipeline Hazard
Situations that prevent pipelining: data hazards (dependency between instructions), control hazards (branch instructions), structural hazards (resource conflicts).
Parallel Processing
Multiple processors/cores executing instructions simultaneously. Types: SIMD (single instruction, multiple data), MIMD (multiple instruction, multiple data).
Multicore Processors
Multiple independent CPUs on a single chip. True parallelism for multi-threaded applications. Shared or separate cache levels.

Artificial Intelligence

Topic 16 · Paper 3 · Machine learning, neural networks, expert systems, and AI ethics

🧠 Core AI Concepts

Turing Test
A machine passes if a human interrogator cannot distinguish between its responses and a human's. Proposed by Alan Turing in 1950.
Machine Learning
Systems that learn from data without being explicitly programmed. Improves performance through experience.
Supervised Learning
Trained on labelled examples (input–output pairs). Learns to predict outputs for new inputs. Used for classification and regression.
Unsupervised Learning
Finds patterns in unlabelled data. Used for clustering (grouping similar items) and dimensionality reduction.
Reinforcement Learning
Agent learns by interacting with an environment. Gets rewards for correct actions, penalties for wrong ones. Used in game AI.

🕸️ Neural Networks

Structure

  • Input layer: receives raw data features
  • Hidden layers: transform and learn features
  • Output layer: produces predictions/classifications
  • Neurons connected by weighted edges

Training

  • Forward pass: compute output from inputs
  • Calculate error (loss function)
  • Backpropagation: adjust weights to reduce error
  • Gradient descent: iterative optimisation

Applications

  • Image recognition (convolutional networks)
  • Natural language processing (transformers)
  • Speech recognition
  • Medical diagnosis

🧩 Expert Systems

Knowledge Base
Contains domain-specific facts and rules (IF-THEN rules). Built by a knowledge engineer interviewing domain experts.
Inference Engine
Applies logical rules from the knowledge base to known facts to derive conclusions. Forward or backward chaining.
Explanation Facility
Explains the reasoning behind a conclusion. Makes the system transparent and easier to trust (unlike neural networks).
Limitations
Cannot learn from new data; only as good as the knowledge base; expensive to build and maintain; brittle outside their domain.

Computational Thinking & Problem-Solving

Topic 17 · Paper 3 · Complexity, tractability, heuristics, and backtracking

📊 Complexity Classes

ClassDescriptionExample
PSolvable in polynomial time by a deterministic algorithmSorting, shortest path (Dijkstra)
NPVerifiable in polynomial time; may require exponential time to solveBoolean satisfiability, TSP
NP-CompleteHardest problems in NP; all NP problems reducible to themTravelling Salesman, knapsack
TractableSolvable in polynomial time; practically feasibleMost graph algorithms
IntractableNo polynomial time solution known; impractical for large inputsTSP (exact solution)

🎯 Problem-Solving Techniques

Heuristic
An approximation method that finds a "good enough" solution when exact solution is too costly. Trades optimality for speed. Example: nearest-neighbour for TSP.
Backtracking
Systematically tries all possible solutions, abandoning a path (backtracking) as soon as it's determined to not lead to a solution. Example: maze solving, Sudoku.
Dynamic Programming
Breaks problem into overlapping subproblems. Stores results to avoid recomputation (memoisation). Example: Fibonacci, shortest path.
Divide & Conquer
Splits a problem into independent subproblems, solves each recursively, and combines results. Example: merge sort, binary search.

🔷 Declarative Programming (Prolog)

Prolog % Facts parent(tom, bob). parent(tom, liz). parent(bob, ann). % Rules grandparent(X, Z) :- parent(X, Y), parent(Y, Z). % Query: ?- grandparent(tom, ann). % Returns: true
Facts
Base-level truths about the problem domain. Stated directly.
Rules
Define relationships: "X is true if Y and Z are true." Use :- (if) syntax.
Queries
Ask questions of the knowledge base. Prolog uses unification and backtracking to find solutions.

λ Functional Programming

Pure Functions
Output depends only on inputs; no side effects (no modifying global state). Same input always gives same output.
Higher-Order Functions
Functions that take other functions as arguments or return functions. Examples: map, filter, reduce.
Immutability
Data cannot be changed once created. New data is created via transformation. Avoids many bugs from shared mutable state.
List Processing
Head (first element) and Tail (rest of list). Recursive processing is natural in functional languages.
Python (functional style) numbers = [1, 2, 3, 4, 5, 6] # map — apply function to each element squares = list(map(lambda x: x**2, numbers)) # [1,4,9,16,25,36] # filter — keep elements matching predicate evens = list(filter(lambda x: x % 2 == 0, numbers)) # [2,4,6] # reduce — fold list into single value from functools import reduce total = reduce(lambda acc, x: acc + x, numbers) # 21
