Paper 1 · Topic 5

Processor Fundamentals

How the CPU works — registers, the fetch-decode-execute cycle, the system bus, interrupts, and Cambridge assembly language. One of the most heavily examined Paper 1 topics.

Paper 1 · AS & A Level ~14% of Paper 1 marks

✦ Learning Objectives

By the end of this topic you should be able to:

  • Name and state the purpose of each CPU register (PC, MAR, MDR, CIR, ACC, SR, IX)
  • Describe every step of the Fetch-Decode-Execute cycle using register notation
  • Explain the role of the address, data, and control buses and the effect of their width
  • Distinguish immediate, direct, indirect, and indexed addressing modes
  • Identify factors affecting CPU performance and explain how each helps
  • Describe what an interrupt is and give hardware and software examples
  • Trace the interrupt handling sequence including saving state to the stack and ISR execution
  • Use the Cambridge 9618 assembly instruction set to write short programs
  • Complete trace tables for assembly programs

1 CPU Registers

Registers are the CPU's own tiny, ultra-fast storage locations — they hold a single word of data (the same width as the data bus) and are accessed in a single clock cycle. They are volatile: values change every few cycles.

Abbr.Full NamePurpose
PC Program Counter Holds the address of the next instruction to be fetched. Incremented during Fetch, before Execute.
MAR Memory Address Register Holds the address of the memory location currently being read from or written to. Connected directly to the address bus.
MDR / MBR Memory Data Register Holds data read from memory, or data about to be written to memory. Connected directly to the data bus.
CIR / IR Current Instruction Register Holds the instruction currently being decoded and executed. Split into an opcode (what to do) and an operand (what to act on).
ACC Accumulator Holds the result of ALU arithmetic and logic operations. The primary working register for most calculations.
SR / PSR Status Register A set of flag bits set by the ALU after each operation: Carry (C), Zero (Z), Negative (N), Overflow (V). Used by conditional jump instructions.
IX Index Register Used in indexed addressing. Effective address = operand + IX. Enables efficient looping over arrays.
Memory tip: PC always points to the next instruction — not the current one. It is incremented during the Fetch stage (before Decode/Execute). If an interrupt occurs, the current PC value is pushed to the stack so execution can resume after the ISR.

2 The System Bus

The system bus is the shared set of electrical connections that carries signals between the CPU, main memory, and I/O devices. It has three logically separate buses:

2.1 Address Bus

Unidirectional — signals travel from CPU to memory/device only. Carries the address of the memory location to be accessed. The width of the address bus determines the maximum amount of directly addressable memory:

Maximum addressable memory
2n locations    (n = number of address bus bits)

A 32-bit address bus can address 232 = 4 GiB of memory. A 64-bit bus can address 16 EiB.

2.2 Data Bus

Bidirectional — data flows in both directions. Carries the actual data being transferred between components. Its width equals the CPU word size (8, 16, 32, or 64 bits). A wider data bus means more data transferred per clock cycle, improving throughput.

2.3 Control Bus

Bidirectional — individual control lines carry synchronisation signals. Key signals include: Read/Write, Clock, Interrupt request, Reset, Bus grant/request. It coordinates activity on the other two buses.

Exam point: Wider address bus = more addressable memory. Wider data bus = more data per cycle. Both increase performance, but for different reasons — be precise about which you mean.

3 The Fetch-Decode-Execute Cycle

The FDE cycle is the sequence the CPU repeats continuously for every instruction. Each repetition takes at least one clock cycle. The steps must be stated with register transfer notation in the exam.

FETCH MAR ← PC — copy the address of the next instruction to MAR
FETCH PC ← PC + 1 — increment PC to point to the following instruction
FETCH MDR ← Memory[MAR] — read the instruction from memory into MDR (address bus carries MAR; data bus returns data)
FETCH CIR ← MDR — copy the instruction from MDR to CIR ready for decoding
DECODE CU decodes CIR — the Control Unit splits the instruction into opcode and operand, determines what action is required
EXECUTE ALU / CU acts — the ALU performs arithmetic or logic, or the CU controls a data transfer or jump; result stored in ACC or memory
Common mistake: Students say the PC holds the address of the current instruction. It holds the address of the next — it was already incremented during Fetch.

3.1 Opcode and Operand

Each instruction in memory is a binary pattern split into two fields:

  • Opcode — identifies what operation to perform (e.g. ADD, LOAD, JUMP). The Control Unit decodes this field.
  • Operand — the data or address the instruction acts on. Its interpretation depends on the addressing mode.

4 Addressing Modes

The addressing mode determines how the operand field of an instruction is interpreted to find the actual data.

Immediate

The operand is the data value itself. No memory access needed — fast.
LDI 42 → ACC ← 42

Direct

The operand is the memory address of the data. One memory read needed.
LDD 200 → ACC ← Mem[200]

Indirect

The operand is the address of a memory location that contains the effective address. Two memory reads needed. Flexible but slow.
LDA 200 → ACC ← Mem[Mem[200]]

Indexed

Effective address = operand + IX register. Used for looping over arrays without rewriting code.
LDX 300 → ACC ← Mem[300 + IX]

Worked Example — Addressing modes compared

Suppose Mem[200] = 50 and Mem[50] = 99, and IX = 5.

1
LDI 200 → ACC = 200 (the number 200 itself)
2
LDD 200 → ACC = 50 (the value stored at address 200)
3
LDA 200 → ACC = 99 (the value stored at address 50, which was found in address 200)
4
LDX 195 → ACC = 50 (the value stored at address 195 + 5 = 200)

5 Factors Affecting CPU Performance

Several hardware characteristics determine how many instructions the CPU can execute per second. You must be able to explain how each factor helps — not just name it.

⏱ Clock Speed

Number of cycles per second (Hz / GHz). More cycles = more FDE cycles per second = faster execution. Typical modern CPU: 3–5 GHz.

🧩 Number of Cores

Each core is an independent processor. Multiple cores allow true parallel execution of separate threads or programs simultaneously.

💾 Cache Size

Cache is very fast SRAM on or near the CPU die. Larger cache stores more frequently-used data, reducing slow main-memory accesses (cache misses).

📏 Word Length

The size of registers and the data bus (e.g. 64-bit). A wider word processes more data per instruction, reducing instructions needed for large values.

🗺 Address Bus Width

Determines the maximum RAM addressable. Larger addressable memory prevents bottlenecks in RAM-intensive programs.

🔁 Pipelining

Overlaps FDE stages of successive instructions: while one instruction executes, the next decodes, and the one after is fetched simultaneously.

Exam tip: "State two ways a CPU can be made to perform better" — always add a brief explanation. "Increase clock speed so more FDE cycles occur per second" scores 2 marks; "increase clock speed" alone usually scores only 1.

6 Interrupts

An interrupt is a signal sent to the CPU by hardware or software to indicate that an event requires immediate attention. The CPU pauses the current task, handles the interrupt, then resumes.

6.1 Types of Interrupt

TypeSourceExamples
Hardware Physical device Keyboard key pressed, I/O transfer complete, printer ready, timer expiry
Software Running program System call to the OS (deliberately), division by zero, illegal memory access (errors)
Maskable Low-priority device Can be temporarily ignored (masked) if the CPU is in a critical section. Most hardware interrupts are maskable.
Non-maskable (NMI) Critical failure Power failure, memory parity error, hardware fault — cannot be ignored under any circumstances.

6.2 Interrupt Handling Sequence

Restore state

  • ISR (Interrupt Service Routine) — a subroutine that handles a specific interrupt. Each interrupt type maps to its ISR's address via the Interrupt Vector Table.
  • Priority — higher-priority interrupts can preempt a running ISR. An NMI preempts everything.
  • Stack — the CPU's state (all registers) is pushed onto the system stack before the ISR runs, and popped back once the ISR finishes, so the original program resumes exactly where it left off.
Why check only at the end of a cycle? Checking mid-instruction would leave registers in an inconsistent state (e.g. only half an addition done). Waiting for the current FDE cycle to complete ensures data integrity before the context switch.

7 Assembly Language (Cambridge 9618)

Cambridge 9618 uses a single-accumulator assembly language. Each instruction has an opcode (mnemonic) and one operand. You must know all mnemonics below.

7.2 Worked Example — Add two inputs

Read two numbers, add them together, output the result.

IN ; ACC ← user input (first number) STO Num1 ; Mem[Num1] ← ACC IN ; ACC ← user input (second number) ADD Num1 ; ACC ← ACC + Mem[Num1] OUT ; output the sum END

7.3 Worked Example — Count down loop

Count from 5 down to 1, outputting each value.

LDI 5 ; ACC ← 5 (counter) STO Count ; Mem[Count] ← 5 LOOP: LDD Count ; ACC ← current counter OUT ; output current value DEC ACC ; ACC ← ACC − 1 STO Count ; save updated counter CMI 0 ; compare counter with 0 JPN LOOP ; if not zero, loop again END

7.4 Trace Tables

A trace table shows the state of all relevant registers/memory locations after each instruction. It is frequently required in exam questions — one row per instruction executed.

Worked Trace — LDI 10 / STO 300 / LDI 3 / ADD 300 / STO 301 / SUB 300 / OUT / END
LineInstructionACCMem[300]Mem[301]Output
1LDI 1010——
2STO 3001010—
3LDI 3310—
4ADD 3001310—
5STO 301131013

8END—1013
Technique: Fill the table left-to-right, row-by-row. Only update a column if the instruction changes that value. Leave unchanged cells carrying the previous value — markers award marks per correct cell.
Key assembly exam points:
  • LDI loads the literal value; LDD loads from an address — do not mix them up.
  • CMP/CMI set flags but do not change ACC.
  • JPE fires when Z=1 (ACC equalled the compared value); JPN fires when Z=0 (not equal — not "negative").
  • LSL 1 multiplies by 2; LSR 1 divides by 2 (integer).

Question 1 · 4 marks

State the purpose of each register: (a) PC   (b) MAR   (c) MDR   (d) ACC

(a) PC: Holds the address of the next instruction to be fetched. [1]

(b) MAR: Holds the address of the memory location currently being read from or written to. [1]

(c) MDR: Holds the data that has been read from memory, or the data to be written to memory. [1]

(d) ACC: Holds the result of ALU arithmetic and logical operations. [1]

Question 2 · 6 marks

Describe the Fetch stage of the FDE cycle, using register transfer notation. [6]

Award 1 mark each (max 6):

  • MAR ← PC — the address of the next instruction is copied to MAR [1]
  • PC ← PC + 1 — PC incremented to point to the following instruction [1]
  • MDR ← Memory[MAR] — instruction fetched from memory into MDR [1]
  • CIR ← MDR — instruction copied from MDR to CIR ready for decoding [1]
  • MAR is placed on the address bus; memory reads via the control bus [1]
  • The instruction returns on the data bus into MDR [1]

Question 4 · 4 marks

(a) Describe what happens when an interrupt is received at the end of an FDE cycle. [3]
(b) Explain why interrupts are only checked at the end of a cycle. [1]

(a) Any 3 of:

  • The CPU checks the interrupt register / flag at the end of the current FDE cycle
  • The current CPU state (PC, ACC, SR, etc.) is saved to the stack
  • The address of the appropriate ISR is loaded into the PC
  • The ISR executes to service the interrupt
  • After the ISR, the saved state is restored from the stack and execution resumes

(b) To ensure the current instruction completes fully — interrupting mid-instruction would leave registers in an inconsistent state, corrupting the program's data.

Question 5 · 4 marks

Write a Cambridge 9618 assembly program that reads a number, doubles it, and outputs the result. [4]

IN ; ACC ← input LSL 1 ; ACC ← ACC × 2 (shift left by 1 bit) OUT ; output doubled value END

Accept alternative using ADD: STO Num / ADD Num / OUT / END

Award: 1 mark IN; 1 mark correct doubling instruction; 1 mark OUT; 1 mark END. [4]

Common Mistakes

  • PC holds the next instruction address — not the current one
  • Confusing LDI (literal value) with LDD (address lookup)
  • CMP does not change ACC — flags only
  • JPN means "not equal" (Z=0), not "negative"
  • Forgetting to save CPU state to the stack before the ISR
  • Wider address bus = more memory; wider data bus = more data per cycle — don't swap these
