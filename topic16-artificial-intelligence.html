1 What is Artificial Intelligence?

Artificial Intelligence (AI) is the field of computer science concerned with building systems that can perform tasks that would normally require human intelligence — reasoning, learning, problem-solving, perception, and language understanding.

The Turing Test

In 1950, Alan Turing proposed a practical test for machine intelligence. In the Turing Test, a human interrogator conducts a text conversation with two participants hidden from view — one is a human, the other a machine. If the interrogator cannot reliably distinguish the machine from the human after a sustained conversation, the machine is said to have passed the test and may be considered intelligent.

Imitation Game

Turing's original framing — the machine "imitates" a human. The interrogator asks any question and judges the replies. The machine wins if the interrogator gets it wrong.

What it measures

The ability to produce convincing natural-language responses. It tests language and reasoning — but not physical ability, emotion, or consciousness.

Criticisms

Passing the test does not mean the machine "understands" anything (Chinese Room argument). A machine could pass by deception rather than genuine intelligence.

Significance

Shifted AI from philosophical "can machines think?" to a practical, measurable question. Still widely referenced as a benchmark concept in AI discussions.

Weak AI vs Strong AI

TypeDefinitionExample
Weak (Narrow) AI A system designed to perform one specific task well. Does not generalise beyond its domain. Has no general reasoning or consciousness. Chess engine, spam filter, image classifier, speech recognition
Strong (General) AI A hypothetical system with human-level general reasoning ability — able to learn and perform any intellectual task a human can. Does not currently exist. Hypothetical: a system that can both play chess, write poetry, and diagnose illness without separate training for each
Exam tip: All current AI systems (including large language models) are weak/narrow AI. Strong AI is a theoretical concept — do not claim it exists. The Cambridge syllabus focuses on weak AI applications.

Knowledge Representation

For a computer to "reason," knowledge must be encoded in a form the system can process. Common representations include:

  • Rules (IF–THEN): IF the patient has a fever AND a cough THEN suspect flu. Used in expert systems.
  • Semantic networks: Nodes representing concepts linked by labelled relationships (e.g., "Dog IS-A Animal").
  • Frames: Structured records for stereotyped situations (similar to objects in OOP).
  • Numeric weights: Used in neural networks — knowledge is implicitly encoded in connection strengths.
Practice Question

Describe the Turing Test and explain one limitation of using it as a measure of machine intelligence.

Turing Test: A human interrogator holds a text conversation with two participants — one human, one machine — without knowing which is which. If the interrogator cannot reliably identify the machine after sustained questioning, the machine is said to pass the test. (2 marks)

Limitation (any one):

  • A machine could pass through clever deception or mimicry without genuinely understanding anything. (1 mark)
  • The test only measures linguistic performance, not broader intelligence such as physical skill, creativity, or emotional understanding. (1 mark)
  • The Chinese Room argument (Searle): a system can produce correct responses purely by symbol manipulation, with no actual comprehension. (1 mark)

2 Machine Learning

Machine learning (ML) is a subset of AI in which a system learns patterns from data without being explicitly programmed with the rules. Instead of the programmer writing the logic, the algorithm infers the logic from examples.

Key Terminology

Training Data

The dataset used to train the model. The algorithm processes examples and adjusts its internal parameters to minimise errors on this data.

Features

The measurable properties (input variables) of each example. For a spam filter: sender, subject line length, presence of certain words.

Label

The correct answer (output) for each training example. In supervised learning only. E.g., "spam" or "not spam" for each email.

Model

The trained system — a mathematical function mapping inputs (features) to outputs (predictions). The result of the training process.

Validation Set

A held-out dataset not used during training. Used to measure how well the model generalises to unseen data and to detect overfitting.

Overfitting

The model learns the training data too precisely — including noise — and performs poorly on new data. High training accuracy but low validation accuracy.

The Three Learning Paradigms

Supervised Learning

The algorithm is given labelled training examples — each input is paired with the correct output. The model learns to map inputs to outputs by minimising the difference between its predictions and the true labels.

AspectDetail
RequiresLabelled dataset (human-annotated)
GoalPredict a label for new, unseen inputs
Sub-tasksClassification — output is a category (e.g., cat/dog, spam/not-spam); Regression — output is a continuous value (e.g., house price)
ExamplesEmail spam filter, image classifier, medical diagnosis, credit scoring
Common algorithmsDecision trees, k-nearest neighbours, neural networks, support vector machines

Unsupervised Learning

The algorithm is given unlabelled data and must discover hidden structure or patterns on its own. There are no correct answers provided.

AspectDetail
RequiresUnlabelled dataset
GoalDiscover patterns, groupings, or structure
Sub-tasksClustering — group similar examples together; Dimensionality reduction — compress data to fewer features while preserving structure
ExamplesCustomer segmentation, anomaly detection, document topic modelling, gene expression analysis
Common algorithmsk-means clustering, hierarchical clustering, principal component analysis (PCA)

Reinforcement Learning

An agent learns by interacting with an environment. It takes actions, receives a reward (positive) or penalty (negative) signal, and updates its strategy (policy) to maximise cumulative reward over time. There is no labelled dataset — the agent discovers what works through trial and error.

ComponentDefinition
AgentThe learning system that takes actions
EnvironmentThe system the agent interacts with
StateThe current situation of the agent in the environment
ActionA choice made by the agent that changes the state
RewardFeedback signal: positive for good actions, negative for bad ones
PolicyThe agent's strategy: maps states to actions to maximise future reward

Examples: Game-playing AI (chess, Go, video games), robot locomotion training, recommendation systems, self-driving vehicle navigation.

Worked Example — Classifying a Learning Task

A streaming service wants to group users into segments based on their watching habits (genre, duration, time of day), without knowing in advance what the groups should be.

1
Is the data labelled? No — there is no pre-assigned "correct group" for each user.
2
Is there an agent interacting with an environment? No.
3
Conclusion: this is unsupervised learning, specifically clustering.

If instead the service wanted to predict whether a user would churn (leave), and had historical data labelled "churned / stayed", that would be supervised classification.

Overfitting vs Underfitting

ProblemCauseSymptomFix
Overfitting Model is too complex; memorises training data noise High training accuracy, low validation accuracy More training data, regularisation, simpler model, dropout
Underfitting Model is too simple; cannot capture the underlying pattern Low training accuracy and low validation accuracy More features, more complex model, longer training
Good fit Model complexity matches data complexity High training accuracy ≈ high validation accuracy Monitor training vs validation loss curves
Key point: A model that achieves 99% accuracy on training data but only 60% on the validation set is overfitting. The solution is not to train longer — training longer usually makes it worse. Collect more diverse data or simplify the model.
Practice Question

A hospital wants to train a model that predicts whether a tumour is malignant or benign, given features such as size, density, and irregular border. A dataset of 10,000 scans has been collected, each labelled by a radiologist as "malignant" or "benign".

(a) Identify the type of machine learning being used. (1 mark)

(b) Explain what is meant by overfitting in this context, and describe one method to detect it. (3 marks)

(a) Supervised learning (classification). The data is labelled (each scan has a correct answer), and the task is to predict a category (malignant/benign). (1 mark)

(b) Overfitting: The model learns the specific patterns of the training scans — including irrelevant noise and coincidences — rather than the genuine features of malignancy. It performs very accurately on training scans it has already seen, but poorly when given new, unseen scans from different patients. (1 mark)

Detection: Split the 10,000 scans into a training set and a separate validation set (held-out data not used during training). After training, measure the accuracy on both sets. If training accuracy is much higher than validation accuracy (e.g., 98% vs 65%), the model is overfitting. (2 marks)

3 Neural Networks

An artificial neural network (ANN) is a computational model loosely inspired by the structure of biological neurons in the brain. It consists of layers of interconnected nodes (neurons) — each connection has a numerical weight that determines how strongly signals are transmitted. Neural networks learn by adjusting these weights.

Hidden layersOne or more layers between input and output. Each neuron computes a weighted sum of its inputs, adds a bias, and applies an activation function. This is where the network learns abstract representations.
Output layerProduces the final prediction. Number of neurons depends on the task: 1 for binary classification, N for N-class classification, 1 for regression.

How a Single Neuron Works

Each neuron in a hidden or output layer performs three steps:

  1. Weighted sum: Multiply each input by its weight and sum them: z = w₁x₁ + w₂x₂ + … + wₙxₙ + b (where b is the bias).
  2. Activation function: Apply a non-linear function f(z) to introduce non-linearity. Without this, the entire network collapses to a linear function regardless of depth.
  3. Output: Pass the result to neurons in the next layer.
Activation FunctionFormulaUse / Properties
Sigmoidσ(z) = 1 / (1 + e⁻ᶻ)Squashes output to (0, 1). Used in output layer for binary classification (gives a probability).
ReLUf(z) = max(0, z)Returns z if positive, else 0. Most common in hidden layers — fast to compute, avoids vanishing gradient.
SoftmaxNormalises outputs to sum to 1Output layer for multi-class classification — gives a probability distribution over all classes.
Tanhtanh(z) = (eᶻ − e⁻ᶻ) / (eᶻ + e⁻ᶻ)Squashes to (−1, 1). Zero-centred, sometimes preferred over sigmoid in hidden layers.

Training: Forward Propagation and Backpropagation

Neural networks learn by iteratively adjusting weights to minimise a loss function — a measure of how wrong the network's predictions are.

1. Forward Propagation

Input data flows through the network layer by layer. Each neuron applies its weights, bias, and activation function. The final layer produces a prediction ŷ.

2. Loss Calculation

Compare the prediction ŷ to the true label y using a loss function (e.g., cross-entropy for classification, mean squared error for regression). A high loss means the prediction was very wrong.

3. Backpropagation

Compute how much each weight contributed to the loss using the chain rule of calculus. The gradient of the loss with respect to each weight is propagated backwards through the network.

4. Gradient Descent

Each weight is updated: w ← w − η × (∂Loss/∂w), where η (eta) is the learning rate. Small η = slow convergence; large η = overshooting. Repeated for many batches of data.

5. Epoch

One complete pass through the entire training dataset. Training typically runs for many epochs — the loss should decrease with each epoch if the model is learning correctly.

6. Convergence

Training stops when the loss stops decreasing significantly, or after a maximum number of epochs. The weights at this point define the trained model.

Worked Example — Tracing Forward Propagation

A simple network has one hidden neuron with inputs x₁ = 1, x₂ = 0.5, weights w₁ = 0.8, w₂ = 0.4, bias b = 0.1. Activation: ReLU.

1
Weighted sum: z = (0.8 × 1) + (0.4 × 0.5) + 0.1 = 0.8 + 0.2 + 0.1 = 1.1
2
Apply ReLU: f(1.1) = max(0, 1.1) = 1.1 (positive, so unchanged)
3
The neuron outputs 1.1, which becomes the input to the next layer.

If z were negative (e.g., −0.3), ReLU would output 0, effectively "switching off" that neuron for this input.

Deep Learning

Deep learning refers to neural networks with many hidden layers (typically 10+). Each layer learns increasingly abstract representations: early layers detect edges and simple patterns; later layers detect complex objects and concepts. Deep networks have achieved state-of-the-art results in image recognition, speech recognition, and natural language processing.

Exam tip: You are not expected to calculate backpropagation by hand. However, you must be able to: (1) describe the purpose of forward propagation, the loss function, and backpropagation in training; (2) explain what weights, biases, and activation functions do; (3) explain why more data and more layers generally improve accuracy (to a point). Always relate these to the learning process.
Practice Question

Describe the role of backpropagation in training a neural network. Include in your answer the terms: loss function, gradient, and weight. (4 marks)

After a forward pass produces a prediction, the loss function measures how far the prediction is from the correct label — a high loss means the network made a large error. (1 mark)

Backpropagation calculates the gradient of the loss with respect to each weight in the network — this tells us how much and in which direction each weight contributed to the error. (1 mark)

The gradients are propagated backwards through the network layer by layer using the chain rule of calculus, from the output layer back to the input layer. (1 mark)

Each weight is then updated in the direction that reduces the loss (gradient descent): the weight is decreased by a small amount proportional to its gradient and the learning rate. Repeating this process over many training examples allows the network to progressively reduce its error and learn the correct patterns. (1 mark)

4 Expert Systems

An expert system is a computer program that emulates the decision-making ability of a human expert in a specific domain. Unlike neural networks, expert systems store knowledge explicitly as human-readable rules and can explain how they reached a conclusion.

Components of an Expert System

Knowledge Base

Contains domain facts and IF-THEN rules provided by human experts. The "memory" of the system.

Inference Engine

Applies rules from the knowledge base to known facts to derive new conclusions. The "brain" of the system.

Explanation Facility

Explains which rules were fired and why a conclusion was reached. Makes the system transparent and auditable.

Additional components often include a user interface (for asking questions and displaying conclusions) and a working memory (the set of currently known facts, updated as the session progresses).

The Knowledge Base

The knowledge base is built by a knowledge engineer who interviews domain experts (e.g., doctors, lawyers, engineers) and encodes their expertise as rules. A typical rule takes the form:

IF patient_temperature > 38.5 AND symptom = "cough" AND symptom = "fatigue"
THEN diagnosis = "possible influenza" WITH confidence = 0.75

Many rules use confidence factors (certainty factors) to handle uncertainty — real-world knowledge is rarely absolute.

The Inference Engine: Forward vs Backward Chaining

MethodDirectionHow it worksUse case
Forward Chaining Data → Conclusion Starts with known facts, fires all applicable rules, adds new facts, and repeats until a conclusion is reached or no more rules can fire. Also called "data-driven." Monitoring systems, process control (e.g., detect all alerts given sensor readings)
Backward Chaining Conclusion → Data Starts with a goal (hypothesis) and works backwards — asks "which rules could prove this?" and then tries to verify the conditions of those rules. Also called "goal-driven." Diagnosis systems (e.g., "Is the patient diabetic?" → verify conditions)
Worked Example — Forward Chaining

Knowledge base:

Rule 1: IF animal_has_fur AND animal_has_claws THEN animal_is_mammal
Rule 2: IF animal_is_mammal AND animal_hunts_prey THEN animal_is_carnivore
Rule 3: IF animal_is_carnivore AND animal_has_stripes THEN animal_is_tiger

Initial facts: animal_has_fur, animal_has_claws, animal_hunts_prey, animal_has_stripes

1
Fire Rule 1 → add animal_is_mammal to working memory.
2
Fire Rule 2 (conditions met) → add animal_is_carnivore.
3
Fire Rule 3 (conditions met) → conclusion: animal_is_tiger.

Advantages and Limitations

AdvantagesLimitations
Can operate 24/7 without fatigue; provides consistent advice Cannot learn from new cases — only as good as the rules in the knowledge base
Can combine knowledge from multiple experts Very expensive and time-consuming to build (knowledge acquisition bottleneck)
Transparent — the explanation facility shows exactly how conclusions were reached Brittle outside the domain — performs poorly on questions outside its knowledge base
Useful in domains where human experts are scarce or unavailable Rules may conflict or become outdated as the domain changes
Can handle uncertainty through confidence factors Cannot handle genuinely novel situations not covered by existing rules
Exam tip: A question asking to compare expert systems with neural networks is common. Key contrasts: (1) expert systems use explicit rules — neural networks encode knowledge implicitly in weights; (2) expert systems can explain their reasoning — neural networks are "black boxes"; (3) expert systems cannot learn from new data — neural networks improve with more training data.
Practice Question

A medical diagnosis system has the following rules in its knowledge base:

Rule A: IF high_blood_pressure AND overweight THEN at_risk_diabetes
Rule B: IF at_risk_diabetes AND high_glucose THEN probable_diabetes
Rule C: IF probable_diabetes AND family_history THEN confirmed_diabetes

The system knows the following facts about a patient: high_blood_pressure, overweight, high_glucose, family_history.

(a) Using forward chaining, state the order in which rules are fired and the final conclusion. (3 marks)

(b) Explain the purpose of the explanation facility in an expert system. (2 marks)

(a)

Fire Rule A: conditions high_blood_pressure AND overweight are both known → adds at_risk_diabetes to working memory. (1 mark)

Fire Rule B: conditions at_risk_diabetes AND high_glucose are both known → adds probable_diabetes. (1 mark)

Fire Rule C: conditions probable_diabetes AND family_history are both known → conclusion: confirmed_diabetes. (1 mark)

(b) The explanation facility records which rules were fired and in what order to reach the conclusion. (1 mark) It allows the doctor (or patient) to understand why the system reached its diagnosis — for example, "diabetes was confirmed because the patient had probable diabetes (from high glucose and being at risk) combined with a family history." This makes the system transparent, auditable, and easier to trust, and allows experts to identify if the reasoning was flawed. (1 mark)

5 Natural Language Processing (NLP)

Natural Language Processing (NLP) is the branch of AI concerned with enabling computers to understand, interpret, and generate human language. Human language is ambiguous, context-dependent, and constantly evolving — making it one of the hardest problems in AI.

Key NLP Tasks

Tokenisation

Splitting text into meaningful units (tokens) — typically words or sub-words. The first step in almost every NLP pipeline.

Example: "The cat sat." → ["The", "cat", "sat", "."]

Part-of-Speech Tagging

Labelling each token with its grammatical role: noun, verb, adjective, etc. Helps parse sentence structure.

Example: "run" (verb) vs "run" (noun) in context.

Parsing (Syntax Analysis)

Building a structured representation of a sentence — a parse tree — showing how words relate grammatically.

Example: Identifying the subject, verb, and object in "The dog chased the cat."

Named Entity Recognition

Identifying proper nouns: people, organisations, locations, dates. Used in information extraction.

Example: "Apple (ORG) was founded in Cupertino (LOC) in 1976 (DATE)."

Sentiment Analysis

Determining the emotional tone of text: positive, negative, or neutral. Used in social media monitoring and product reviews.

Example: "The film was absolutely brilliant!" → positive.

Machine Translation

Automatically translating text from one natural language to another. Modern systems use neural networks (transformer models).

Example: "Bonjour le monde" → "Hello the world."

Challenges of NLP

ChallengeExplanationExample
Ambiguity Words and sentences often have multiple meanings. Context is required to choose the correct interpretation. "I saw her duck" — did she lower her head, or did I see a duck she owns?
Coreference Pronouns and references must be resolved to identify what entity they refer to. "Alice told Bob she was tired" — who was tired, Alice or Bob?
Context and world knowledge Understanding language often requires background knowledge not stated in the text. "The trophy didn't fit in the suitcase because it was too big." — what is too big?
Idioms and figurative language Phrases whose meaning cannot be inferred from the literal meaning of individual words. "It's raining cats and dogs" cannot be understood word-for-word.
Language variation Dialects, slang, spelling errors, and evolving vocabulary make parsing unreliable. "gonna", "tbh", "gr8" are not standard dictionary words.

A Brief History of NLP Approaches

NLP has evolved through three broad eras:

  1. Rule-based (1950s–1980s): Hand-coded grammar rules and dictionaries. Brittle — failed on text outside the grammar's scope.
  2. Statistical (1990s–2000s): Trained on large corpora using probabilistic models. More robust, but required extensive feature engineering.
  3. Neural (2010s–present): Deep learning models (RNNs, LSTMs, Transformers) learn representations directly from text. Modern large language models (e.g., GPT, BERT) achieve near-human performance on many NLP benchmarks.
Key insight: Modern NLP succeeds partly because the internet provides vast quantities of human-written text for training. The more diverse and large the training corpus, the better the model captures the subtleties of language.
Practice Question

Explain two reasons why natural language processing is a challenging problem for computers, compared to processing formal languages such as programming languages. (4 marks)

Reason 1 — Ambiguity: Words and sentences in natural language can have multiple meanings. A computer must use context to determine the correct interpretation, but this context may itself be ambiguous. Programming languages are unambiguous by design — every syntactically valid statement has exactly one defined meaning. (2 marks)

Reason 2 — World knowledge and context: Understanding natural language often requires general knowledge about the world that is not stated in the text (e.g., knowing that suitcases are smaller than trophies helps resolve "it was too big"). Programming languages require no world knowledge — they follow strict syntactic rules the computer can check mechanically. (2 marks)

Also accept: idioms/figurative language, coreference resolution, spelling/grammar variation, sarcasm, evolving slang.

📝 Exam Practice

Question 1 — 4 marks

Compare supervised learning and unsupervised learning. Your answer should include: the type of data required, the goal of each approach, and one example of each.

Supervised learning requires labelled data — each training example has an input and a correct output (label). The goal is to learn a function that maps inputs to labels, so the model can predict labels for new, unseen inputs. (1 mark) Example: Training an email spam classifier using thousands of emails labelled "spam" or "not spam." (1 mark)

Unsupervised learning uses unlabelled data — no correct outputs are provided. The goal is to discover hidden structure or patterns in the data (e.g., groups, clusters, or compressed representations). (1 mark) Example: Grouping customers into segments based on purchase history, without knowing in advance what the groups should be. (1 mark)

Question 2 — 5 marks

A game-playing AI learns to play a video game by trying different actions and receiving a score after each game. The AI is not given any rules about the game — it must discover what to do on its own.

(a) Identify the type of machine learning being used. Justify your answer. (2 marks)

(b) Define the terms agent, state, and reward as they apply to this scenario. (3 marks)

(a) Reinforcement learning. (1 mark) The AI (agent) learns by interacting with the environment (the game) and receiving feedback (score/reward) — it is not given labelled examples (not supervised) nor is it finding structure in unlabelled data (not unsupervised). It discovers optimal actions through trial and error. (1 mark)

(b)

  • Agent: The AI program that plays the game — it observes the game state and selects actions. (1 mark)
  • State: The current situation of the game — for example, the position of the player, enemies, and score at a given moment. (1 mark)
  • Reward: The score (or change in score) received after each action or after each game — a positive reward for good actions (e.g., killing an enemy), a negative reward (penalty) for bad actions (e.g., losing a life). (1 mark)
Question 3 — 5 marks

Describe the structure of a neural network trained to classify handwritten digits (0–9) from 28×28 pixel images. Your answer should include the number of input and output neurons and the roles of the hidden layers. Then explain how the network is trained using backpropagation. (5 marks)

Structure: The input layer has 784 neurons (one per pixel in the 28×28 image — each pixel's greyscale value is a feature). (1 mark) The output layer has 10 neurons, one per digit class (0–9) — a softmax activation produces a probability distribution, and the digit with the highest probability is the prediction. (1 mark) Hidden layers in between transform the raw pixel values into increasingly abstract representations — early layers may detect edges and curves; later layers recognise digit-specific shapes. (1 mark)

Training (backpropagation): A training image is passed through the network (forward propagation) to produce a predicted probability distribution. The loss function (cross-entropy) measures how far the prediction is from the true label (e.g., predicted 70% for "3" but the true digit was "8"). (1 mark) Backpropagation calculates the gradient of the loss with respect to every weight, propagating the error signal backwards through the network. Gradient descent updates each weight to reduce the loss. This is repeated for many images over many epochs until the loss converges. (1 mark)

Question 4 — 6 marks

Compare expert systems and neural networks as approaches to building an AI system for medical diagnosis. Your answer should discuss: knowledge acquisition, transparency, learning from new data, and one advantage unique to each approach.

Knowledge acquisition: Expert systems require a knowledge engineer to interview human doctors and encode their knowledge as explicit IF-THEN rules — this is expensive and time-consuming (knowledge acquisition bottleneck). Neural networks learn automatically from large datasets of labelled patient records — no manual rule creation is needed, but a large, labelled dataset must exist. (1 mark)

Transparency: Expert systems are transparent — the explanation facility can show exactly which rules fired to reach a diagnosis (e.g., "confirmed diabetes because blood glucose was high AND the patient was at risk"). Neural networks are "black boxes" — it is very difficult to explain why a particular diagnosis was made, which is a serious concern in safety-critical medical applications. (2 marks)

Learning from new data: Expert systems cannot update their knowledge automatically — if a new treatment protocol is discovered, the knowledge base must be updated by a knowledge engineer. Neural networks can be re-trained on new data, potentially improving over time as more patient records become available. (1 mark)

Advantage of expert systems: Can operate reliably even with small datasets — rules work with even one example if the expert can articulate the rule. (1 mark)

Advantage of neural networks: Can discover subtle, complex patterns in data (e.g., rare combinations of symptoms) that human experts may not be able to articulate as rules. (1 mark)

Question 5 — 3 marks

Give three challenges that make natural language processing more difficult than processing a formal programming language.

Any three of:

  • Ambiguity: Words and sentences in natural language can have multiple valid interpretations; context is needed to disambiguate. Programming languages are unambiguous by design. (1 mark)
  • Coreference resolution: Pronouns must be resolved to their referents, which can be unclear. "She told her to leave" — who told whom? (1 mark)
  • Idioms and figurative language: Phrases like "kick the bucket" cannot be understood word-for-word. Programming languages have no idioms. (1 mark)
  • World knowledge: Understanding often requires background knowledge not in the text itself. Programming languages need no external knowledge. (1 mark)
  • Variation and evolution: Natural language includes dialects, slang, abbreviations, and spelling errors. Programming languages have fixed, strict syntax. (1 mark)
Question 6 — 4 marks

Explain what is meant by the Turing Test and discuss whether passing the Turing Test should be considered sufficient evidence that a machine is truly intelligent.

Turing Test: A human interrogator conducts a text-based conversation with both a human and a machine, without knowing which is which. If the interrogator cannot reliably distinguish the machine from the human, the machine passes the test. Turing proposed this as a practical substitute for the unanswerable question "can machines think?" (2 marks)

Argument against sufficiency: Passing the Turing Test does not necessarily mean a machine is truly intelligent or understands anything. The Chinese Room argument (Searle) illustrates this — a system could manipulate symbols according to rules and produce convincing replies with no genuine comprehension. Additionally, the test only measures linguistic performance, not creativity, emotion, physical intelligence, or moral reasoning — aspects of human intelligence not captured by text alone. A machine might also pass through deception (e.g., deliberately making typos) rather than genuine understanding. (2 marks)

  • Sigmoid — (0,1) — binary output
  • Softmax — sums to 1 — multi-class output

Common Mistakes

  • All current AI is weak/narrow AI — do not claim strong AI exists
  • Passing the Turing Test ≠ genuine understanding — it tests behaviour, not cognition
  • Neural networks cannot explain their reasoning — expert systems can
  • Expert systems do not learn from new data — knowledge must be manually updated
  • Overfitting: high training accuracy + low validation accuracy — don't confuse with underfitting
