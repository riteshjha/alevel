Topic 4 – Logic Gates & Boolean Algebra

Gates, truth tables, simplification laws, K-maps, and circuit diagrams.

Paper 1 AS Level ~3 exam marks / section

1. Logic Gates

A logic gate is an electronic circuit with one or more binary inputs and a single binary output. The output is fully determined by the inputs according to a Boolean function.

AND

⊓

Output 1 only when all inputs are 1

OR

⊔

Output 1 when at least one input is 1

NOT

Inverts the input (one input only)

NAND

⊼

AND then invert — universal gate

NOR

⊽

OR then invert — universal gate

XOR

⊕

Output 1 when inputs differ

Exam tip: The bubble (small circle) on the output means invert. NAND = AND + bubble; NOR = OR + bubble. If you forget a symbol, drawing the equivalent gates joined together also earns full marks.

3. Boolean Notation

OperationNotationAlternative
ANDA · BAB or A AND B
ORA + BA OR B

Operator Precedence

Evaluate in this order (highest to lowest):

  1. Parentheses — innermost first
  2. NOT (Ā)
  3. AND (·)
  4. OR (+)
Example: A + B · C̄ means A + (B · (NOT C)) — AND before OR.

4. Boolean Algebra Laws

These identities let you simplify expressions without drawing a truth table.

Identity & Null Laws

Identity
A + 0 = A  |  A · 1 = A
Null / Annihilator
A + 1 = 1  |  A · 0 = 0
Idempotent
A + A = A  |  A · A = A
Complement
A + Ā = 1  |  A · Ā = 0
Double Negation
Ā̄ = A

Commutative & Associative

Commutative
A + B = B + A  |  A · B = B · A
Associative
(A+B)+C = A+(B+C)
(AB)C = A(BC)
Distributive
A(B+C) = AB + AC
A+BC = (A+B)(A+C)

De Morgan's Theorems ★

De Morgan 1
(A · B)' = Ā + B̄
NAND = NOR of complements
De Morgan 2
(A + B)' = Ā · B̄
NOR = AND of complements
Memory trick for De Morgan's: Break the bar, change the operator. (A·B)' → Ā + B̄ (bar breaks, dot becomes plus). (A+B)' → Ā · B̄ (bar breaks, plus becomes dot).

Absorption Laws

Absorption 1
A + AB = A
Absorption 2
A(A + B) = A
Useful identity
A + ĀB = A + B

5. Simplifying Boolean Expressions

Simplifying reduces the number of gates needed, lowering cost, power, and propagation delay.

Worked Example 1

Simplify F = AB + AB̄

F = AB + AB̄Given
= A(B + B̄)Factor out A
= A · 1Complement law: B + B̄ = 1
= AIdentity law: A · 1 = A
Result: The circuit needs only a wire — no gates at all!

Worked Example 2

Simplify F = ABC + ABC̄ + AB̄C

= AB(C + C̄) + AB̄CFactor AB from first two terms
= AB · 1 + AB̄CComplement law
= AB + AB̄CIdentity
= A(B + B̄C)Factor A
= A(B + C)Absorption: B + B̄C = B + C

Worked Example 3 — De Morgan's

Simplify F = (A + B)' · C + A · C

= (Ā · B̄) · C + ACDe Morgan on (A+B)'
= ĀB̄C + ACRemove redundant brackets
= C(ĀB̄ + A)Factor C
= C(A + B̄)Absorption: ĀB̄ + A = A + B̄
Exam strategy: Show every step with the law name. Even if your final answer is wrong you still score for correct intermediate steps.

6. Logic Circuit Diagrams

A logic circuit is a diagram connecting gate symbols with lines representing signal wires. You need to:

  • Draw a circuit from a Boolean expression
  • Write a Boolean expression from a circuit
  • Trace a circuit to produce a truth table

Worked Example — Draw from Expression

Expression: Q = (A · B) + (Ā · C)

Circuit for Q = (A · B) + (Ā · C)  — note the junction dot on A feeding both AND and NOT

Junction dots: A filled dot (•) on crossing wires means they are connected. No dot means wires cross but are not connected. Always draw dots at branch points.

Reading an Expression from a Circuit

Start at the output gate and work backwards, substituting each gate's inputs:

  1. Label each gate's output with its function (e.g. G1 = A · B)
  2. At the final gate, write the full expression using the labelled sub-expressions
  3. Simplify if asked

Half Adder

A half adder adds two 1-bit numbers to give a Sum and a Carry output. It is a key application of logic gates in arithmetic circuits.

Half Adder
ABSum (S)Carry (C)
0000
0110
1010
1101

S = A ⊕ B  (XOR)     C = A · B  (AND)

Two half adders joined together (with the carry-ins combined via an OR gate) form a full adder, which can handle a carry-in from a previous stage, allowing multi-bit addition.

7. Karnaugh Maps (K-maps)

A K-map is a visual tool for simplifying Boolean expressions. Adjacent cells differ by exactly one variable (Grey code ordering). Grouping 1s in powers of 2 eliminates variables.

Why K-maps? Algebraic simplification requires intuition and can miss the minimum form. K-maps give a systematic method that always finds the minimal Sum of Products (SOP).

2-Variable K-map

B=0B=1
A=0m₀m₁
A=1m₂m₃

3-Variable K-map

A \ BC00011110
0m₀m₁m₃m₂
1m₄m₅m₇m₆

Note: columns go 00→01→11→10 (Grey code). Column 11 and column 10 are also adjacent to column 00 (the map wraps).

All four 1s form a single group of 4. The column header changes from 01→11, so B varies — B is eliminated. A also varies across rows — A is eliminated. Only C stays constant at 1.

F = CGroup of 4, only C=1 invariant

K-map Rules

Group size
Must be a power of 2: 1, 2, 4, 8…
Shape
Groups must be rectangular (no diagonals)
Wrap-around
Edges of the K-map are adjacent (it wraps)
Largest first
Always make groups as large as possible
Overlap
Groups can overlap — a 1 can be in multiple groups
Don't-cares
X (don't-care) can be 0 or 1 — use them to grow groups
Reading the simplified term: For a group, identify which variables are constant across all cells. If a constant variable is 1, include it uncomplemented; if 0, include it complemented. Variables that change are eliminated.

8. NAND & NOR as Universal Gates

Any Boolean function can be built using only NAND gates (or only NOR gates). This is useful because a single gate type simplifies manufacturing.

Building AND, OR, NOT from NAND

Gate neededNAND equivalent
NOT ANAND(A, A)
A AND BNAND( NAND(A,B), NAND(A,B) )
A OR BNAND( NAND(A,A), NAND(B,B) )

Building AND, OR, NOT from NOR

Gate neededNOR equivalent
NOT ANOR(A, A)
A OR BNOR( NOR(A,B), NOR(A,B) )
A AND BNOR( NOR(A,A), NOR(B,B) )
Proof that NAND gives NOT: NAND(A,A) outputs (A·A)' = A' (using idempotent then complement). One gate, one input duplicated.

(b) State one advantage of simplifying a Boolean expression before building the circuit.   [1]

F = ĀBC + ABC + AB̄C

= BC + AB̄C   [identity]

= C(B + AB̄)   [factor C]

= C(B + A)   [absorption: B + AB̄ = B + A]

Question 3 [6 marks]

The Boolean expression for a logic circuit is: F = (A + B) · (Ā + C)

(a) Draw the logic circuit for this expression.   [3]

(b) Apply De Morgan's theorem to expand and then simplify the expression as far as possible.   [3]

Show mark scheme

(a) Circuit features (1 mark each):

  • NOT gate with input A producing Ā
  • OR gate 1: inputs A and B; OR gate 2: inputs Ā and C
  • AND gate combining both OR outputs to give F — correct connections

(b)

F = (A + B)(Ā + C)

= AĀ + AC + ĀB + BC   [expand (distributive)]

= 0 + AC + ĀB + BC   [complement: AĀ = 0]

= AC + ĀB + BC

Now use consensus: BC = ABC + ĀBC; note AC absorbs ABC and ĀB absorbs ĀBC:

F = AC + ĀB   [1 mark expand, 1 mark eliminate AĀ, 1 mark correct final form]

Key Boolean Laws

  • A·0=0, A+1=1
  • A·1=A, A+0=A
  • A·A=A, A+A=A
  • A·Ā=0, A+Ā=1
  • Ā̄=A
  • De M.: (AB)'=Ā+B̄
  • De M.: (A+B)'=ĀB̄

Gate Quick Ref

  • AND: all 1 → 1
  • OR: any 1 → 1
  • NOT: invert
  • NAND: AND + invert
  • NOR: OR + invert
  • XOR: different → 1

Common Exam Mistakes

  • Forgetting to show law names when simplifying
  • Wrong Grey code order in K-map columns (must be 00,01,11,10)
  • Diagonal groups in K-maps (invalid — must be rectangular)
  • Missing junction dots on circuit diagrams
  • Applying De Morgan's incorrectly — must break the entire bar at once
