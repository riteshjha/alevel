1 Computer Architecture

A computer's architecture defines how its components are organised and how they communicate. The two models you need to know are Von Neumann and Harvard.

1.1 Von Neumann Architecture

Proposed by John von Neumann in 1945, this is the basis of virtually all modern general-purpose computers. Its defining feature is the stored-program concept: instructions and data are stored together in the same main memory, using the same bus to fetch both.

⚠️ The Von Neumann Bottleneck: Because only one thing can travel on the shared bus at a time, the CPU often has to wait for the next instruction or data from memory. This mismatch between fast CPU speed and slower memory access is the Von Neumann bottleneck. It is partially addressed by cache memory.

1.2 Harvard Architecture

The Harvard architecture uses separate memory stores and buses for instructions and data, allowing both to be fetched simultaneously. This eliminates the Von Neumann bottleneck.

🔵 Von Neumann

  • Single shared memory for instructions and data
  • Single bus (address + data + control)
  • Simpler and cheaper design
  • Flexible: RAM can hold any mix of code and data
  • Subject to the Von Neumann bottleneck
  • Used in most general-purpose computers (PCs, laptops)

🔴 Harvard

  • Separate memory for instructions and data
  • Separate buses — can fetch instruction and data simultaneously
  • More complex and expensive hardware
  • No bottleneck — higher throughput
  • Instructions often in read-only memory (ROM)
  • Used in microcontrollers, DSPs, CPUs with separate instruction cache
💡 Modern CPUs are a hybrid: Most modern processors use a modified Harvard architecture internally — they have separate L1 instruction cache and L1 data cache — but present a Von Neumann interface to the programmer (a single unified address space for instructions and data).

1.3 CPU Components

ComponentRole
Arithmetic Logic Unit (ALU) Performs all arithmetic operations (add, subtract, multiply, divide) and logical operations (AND, OR, NOT, comparisons). Results are stored in the accumulator.
Control Unit (CU) Manages and coordinates all CPU operations. Decodes instructions, generates control signals, directs data flow between components, and manages the Fetch-Decode-Execute cycle.
Registers Tiny, extremely fast storage locations inside the CPU. Hold data currently being processed. Examples: PC, MAR, MDR, CIR, ACC, SR. (Covered in detail in Topic 5.)
Clock Generates regular pulses (the clock signal) that synchronise all CPU operations. One FDE cycle takes one or more clock cycles. Measured in GHz (billions of pulses per second).
✏ Practice Questions

(a) State two differences between Von Neumann and Harvard architectures.
(b) Explain what is meant by the Von Neumann bottleneck.
(c) State one application where Harvard architecture is preferred over Von Neumann, and explain why.

(a) Any two: (1) Von Neumann uses a single shared memory for both instructions and data; Harvard uses separate memories. (2) Von Neumann has a single shared bus; Harvard has separate buses for instructions and data. (3) Harvard can fetch an instruction and data simultaneously; Von Neumann cannot.

(b) The CPU can process instructions much faster than data can be fetched from main memory via the single shared bus [1]. The CPU must wait (stall) while memory access completes, limiting overall performance [1].

(c) Example: a microcontroller in an embedded system (e.g. washing machine controller) [1]. Harvard is preferred because the program (stored in ROM) and data (sensor readings) can be fetched simultaneously, giving faster, more predictable performance which is critical in real-time control applications [1].

✏ Practice Questions

(a) Explain the difference between the address bus and the data bus.
(b) A computer has a 16-bit address bus. Calculate the maximum amount of memory that can be addressed, giving your answer in KiB.
(c) Explain why the address bus is unidirectional but the data bus is bidirectional.

(a) The address bus carries the memory address (location) that the CPU wants to access [1]; the data bus carries the actual data or instruction being read from or written to that address [1].

(b) 2¹⁶ = 65,536 bytes = 64 KiB [2]

(c) The address bus is unidirectional because only the CPU generates memory addresses — memory never needs to tell the CPU which address to use [1]. The data bus is bidirectional because data flows both ways: from memory to CPU when reading (load), and from CPU to memory when writing (store) [1].

3 Memory

3.1 Memory Hierarchy

Computer systems use a hierarchy of different memory types, trading off speed, size, and cost. Faster memories are small and expensive; larger memories are slower and cheaper. The CPU works from the top of the hierarchy downwards.

Registers Bytes ~1 cycle · fastest
L1 Cache 32–64 KB ~4 cycles
L2 Cache 256 KB – 1 MB ~12 cycles
L3 Cache (shared) 4–32 MB ~40 cycles
Main Memory (RAM) 8–64 GB ~100 cycles
Secondary Storage (SSD / HDD) 256 GB – 4 TB+ ms–μs · slowest

↑ Faster, smaller, more expensive     ↓ Slower, larger, cheaper

3.2 RAM and ROM

DRAM vs SRAM

DRAM (Dynamic RAM)
Uses a capacitor per bit — must be refreshed thousands of times per second. Slower, cheaper, higher density. Used for main memory.
SRAM (Static RAM)
Uses a flip-flop per bit — no refresh needed. Much faster, more expensive, lower density. Used for cache memory inside the CPU.

3.3 Virtual Memory

When a computer runs more programs than fit in physical RAM, the operating system uses an area of the hard disk or SSD as an extension of RAM. This is called virtual memory.

✎ How Virtual Memory Works
1The OS divides memory into fixed-size blocks called pages (typically 4 KB each).
2When RAM is full and a new page is needed, the OS identifies a page in RAM that hasn't been used recently.
3That page is swapped out to a dedicated area on the disk (called the page file or swap space).
4The required new page is loaded into the freed RAM space.
5If a swapped-out page is needed again, another swap occurs. Excessive swapping is called thrashing.
Virtual memory allows more programs to run than physical RAM alone would permit, but at the cost of performance (disk access is thousands of times slower than RAM).
⚠️ Thrashing: If too little RAM is installed, the system spends more time swapping pages in and out than actually executing program instructions. The computer becomes extremely slow — this is thrashing. Adding more RAM is the solution.
✏ Practice Questions

(a) Explain why ROM is used to store the computer's BIOS rather than RAM.
(b) Describe what happens when virtual memory is used and explain one disadvantage.
(c) State the difference between DRAM and SRAM and explain why SRAM is used for cache.

(a) ROM is non-volatile — it retains its contents when power is off [1]. The BIOS must be available immediately when the computer is switched on, before RAM can be loaded with any program [1]. If the BIOS were in RAM it would be lost when the computer was turned off [1].

(b) When physical RAM is full, the OS moves unused pages from RAM to a page file on the hard disk [1], freeing RAM for the required pages. Disadvantage: accessing the hard disk/SSD is thousands of times slower than RAM [1], so the system runs much more slowly when virtual memory is in heavy use (which can lead to thrashing) [1].

(c) DRAM uses capacitors and must be refreshed thousands of times per second — it is slower and cheaper [1]. SRAM uses flip-flops, requires no refreshing, and is much faster — but is more expensive and less dense [1]. SRAM is used for cache because speed is the priority — cache must supply data to the CPU faster than DRAM can [1].

4 Cache Memory

Cache memory is a small, very fast SRAM memory that sits between the CPU and main memory. It stores copies of data and instructions that have been recently used or are likely to be needed soon. The purpose is to reduce the number of slow main-memory accesses.

4.1 Cache Levels

LevelLocationSizeSpeedShared?
L1 CacheInside the CPU core32–128 KBFastest (~4 CPU cycles)Per core — not shared
L2 CacheInside or very close to CPU core256 KB – 4 MBFast (~12 cycles)Usually per core
L3 CacheOn the CPU chip, outside cores4–64 MBSlower (~40 cycles)Shared across all cores

4.2 Cache Hit and Cache Miss

CPU needs data from address 0x4A2F: 1. CPU checks L1 cache first → CACHE HIT: data found in L1 → returned to CPU in ~4 cycles ✓ → CACHE MISS: not in L1 → check L2... 2. CPU checks L2 cache → CACHE HIT: data found in L2 → returned in ~12 cycles ✓ → CACHE MISS: not in L2 → check L3... 3. CPU checks L3 cache → CACHE HIT: data found in L3 → returned in ~40 cycles ✓ → CACHE MISS: not in any cache... 4. Fetch from main memory (RAM) → Data returned in ~100 cycles (much slower) → A copy is placed in L1/L2/L3 for future use
Temporal Locality
Data used recently is likely to be used again soon. Cache exploits this by keeping recently accessed data. Example: loop variables accessed thousands of times.
Spatial Locality
Data near recently accessed data is likely to be used soon. Cache loads a whole block (cache line) at a time, not just one byte. Example: array elements stored consecutively.
Cache Hit Rate
Percentage of memory accesses satisfied by cache. A high hit rate (>95%) means few slow main-memory accesses. Affected by cache size and the program's access patterns.
Cache Replacement Policy
When cache is full, old data must be evicted. Common policies: LRU (Least Recently Used), FIFO (First In First Out), random replacement.
✏ Practice Questions

(a) Explain why cache memory improves CPU performance.
(b) Describe what happens when a cache miss occurs at all cache levels.
(c) Explain the term temporal locality and state how cache exploits it.

(a) Cache stores copies of frequently used data and instructions in fast SRAM [1]. The CPU can access this data much faster than going to main memory (DRAM) [1], reducing the number of slow memory accesses and keeping the CPU supplied with data without stalling [1].

(b) If data is not found in L1, L2, or L3 cache, the CPU must fetch the data from main memory (RAM) [1], which is much slower (~100+ cycles) [1]. A copy of the data is then stored in the cache levels so future accesses to the same data will be faster (cache hit) [1].

(c) Temporal locality: recently accessed data is likely to be accessed again soon [1]. Cache exploits this by keeping a copy of recently used instructions and data in fast SRAM [1], so when the CPU needs the same data again (e.g. in a loop), it can be read from cache instead of slow main memory [1].

5 Secondary Storage

Secondary storage provides permanent, non-volatile data storage. Unlike RAM, it retains data when power is removed, making it suitable for long-term storage of files, programs, and the operating system.

💿 HDD (Hard Disk Drive)
  • Very large capacity (1–20 TB)
  • Cheap per GB
  • Long-established technology
  • Slow (moving parts, ms access)
  • Fragile — shock sensitive
  • Noisy
  • High power consumption
Technology: Magnetic platters, read/write head moves mechanically
🔷 SSD (Solid State Drive)
  • Fast (μs access time)
  • Silent, no moving parts
  • Light, shock-resistant
  • Low power consumption
  • More expensive per GB than HDD
  • Limited write cycles (wear)
Technology: NAND flash memory cells — no mechanical parts
📀 Optical (CD/DVD/Blu-ray)
  • Cheap media
  • Portable and removable
  • Long shelf life (read-only)
  • Very slow access
  • Low capacity (700 MB–100 GB)
  • Easily scratched
  • Declining use
Technology: Laser reads/writes pits and lands on a reflective disc
📼 Magnetic Tape
  • Extremely high capacity (TB)
  • Very cheap per GB
  • Durable for archiving
  • Sequential access only — very slow random access
  • Not suitable for everyday use
Technology: Magnetic tape. Used for large-scale backups and archiving.

How a Hard Disk Drive Works

Data is stored magnetically on spinning metal platters. The read/write head moves radially while the platter spins, allowing any location to be reached. Access time = seek time (head movement) + rotational latency (waiting for correct sector to spin under head).

Choosing the Right Storage

ScenarioBest ChoiceReason
OS and applications on a laptopSSDFast boot/load times, silent, shock-resistant for portable use
Large video archive in a studioHDDHighest capacity per pound; access speed acceptable for streaming
Daily server backups in a data centreMagnetic TapeExtremely cheap per TB; sequential backup is fine; long archival life
Distributing software to customersOptical (DVD)Cheap to replicate; customers can use without needing a drive (declining)
✏ Practice Questions

(a) A hospital needs to archive patient scans for 30 years at minimum cost. Recommend a storage technology and justify your answer.
(b) Describe two advantages and two disadvantages of SSD compared to HDD.
(c) Explain why magnetic tape is only suitable for sequential access.

(a) Magnetic tape [1]. Justification: very high capacity and extremely low cost per GB for long-term archiving [1]; data is rarely accessed randomly — backups are written sequentially and only retrieved in emergency, so sequential-only access is acceptable [1]; tape has a long archival life of 30+ years when stored correctly [1].

(b) Advantages: (1) Much faster access times (microseconds vs milliseconds for HDD). (2) Silent — no moving parts. (3) More durable/shock-resistant — suitable for portable devices. (4) Lower power consumption [any 2, 1 mark each].
Disadvantages: (1) More expensive per GB than HDD. (2) Limited number of write cycles before cells degrade (wear). [any 2, 1 mark each]

(c) Magnetic tape is wound on a reel — to reach a specific piece of data, the tape must be wound forward or backward through all the data before it [1]. There is no way to jump directly to a random location without physically moving the tape past all the intervening data [1].

7 Factors Affecting CPU Performance

Four main factors determine how fast a CPU executes programs. These are common exam topics — you need to explain each one clearly.

⏱ Clock Speed (GHz)

Number of clock pulses per second. Each pulse allows one or more operations. Doubling clock speed approximately doubles performance (all else equal). Modern CPUs: 3–5+ GHz. Increasing speed also increases heat and power consumption.

🔢 Number of Cores

Each core is an independent processing unit. A quad-core CPU can execute four instruction streams simultaneously. Benefits only multi-threaded software. Adding cores does not help single-threaded programs.

📦 Cache Size

Larger cache can store more recently-used data. Fewer cache misses → fewer slow main-memory accesses → CPU stalls less often. However, larger caches have greater latency — a balance is struck at each level.

📏 Word Length (bits)

The number of bits the CPU processes in one operation (32-bit or 64-bit). A 64-bit CPU can address more memory and process larger numbers in one instruction. Also determines data bus width and register size.

✎ Worked Example — Comparing two CPUs

CPU A: 3.2 GHz, 4 cores, 8 MB L3 cache, 64-bit  vs  CPU B: 4.5 GHz, 2 cores, 4 MB L3 cache, 64-bit

1CPU B has higher clock speed — better for single-threaded tasks (gaming, older software).
2CPU A has more cores — better for multi-threaded tasks (video rendering, running multiple programs).
3CPU A has larger L3 cache — fewer cache misses, better for memory-intensive workloads.
Neither is universally better — the right choice depends on the workload. Always relate performance factors to the specific use case.
✏ Practice Questions

(a) Explain why increasing the number of cores does not always improve the speed of a program.
(b) A video editing workstation is being upgraded. Discuss which CPU performance factor would be most beneficial to improve.
(c) Explain why a 64-bit processor can address more memory than a 32-bit processor.

(a) To benefit from multiple cores, the program must be written to use multiple threads — it must be able to split its work into independent parallel tasks [1]. If a program is single-threaded, additional cores sit idle because the program can only execute on one core at a time [1].

(b) Video editing is a multi-threaded workload — encoding video involves processing many frames simultaneously [1]. Therefore increasing the number of cores would be most beneficial, as each core can encode a different portion of the video in parallel, significantly reducing render time [1]. A larger cache would also help since video editing accesses large amounts of data repeatedly [1].

(c) The number of addressable memory locations = 2n where n is the number of address bits (word length) [1]. A 32-bit processor: 2³² = 4 GiB maximum addressable RAM. A 64-bit processor: 2⁶⁴ ≈ 16 EiB — vastly more [1].

8 Virtual Machines

A virtual machine (VM) is a software emulation of a complete computer. It behaves exactly like a real physical computer — it has its own virtual CPU, RAM, hard disk, and network interface — but it is running entirely within software on a physical host machine.

8.1 The Hypervisor

A hypervisor (also called a Virtual Machine Monitor, VMM) is the software layer that creates and manages virtual machines. It allocates physical hardware resources (CPU time, RAM, disk, network) to each VM.

8.2 Benefits of Virtual Machines

Server Consolidation

  • Multiple servers consolidated onto fewer physical machines
  • Each service runs in its own isolated VM
  • Reduces hardware costs and data centre space
  • Better utilisation of expensive physical hardware

Isolation & Security

  • Each VM is sandboxed — a crash or malware in one VM cannot affect others
  • Ideal for running untrusted software safely
  • Security researchers use VMs to analyse malware

Snapshots & Recovery

  • VM state can be saved as a snapshot at any point
  • Instantly rollback to a known-good state if something goes wrong
  • Simplifies backup and disaster recovery

Portability & Development

  • A VM is just a set of files — easily moved to another physical host
  • Developers can test software on multiple OS versions simultaneously
  • Cloud computing is built on virtualisation (AWS, Azure, GCP)

Legacy Software

  • Run old operating systems (e.g. Windows XP) inside a VM on modern hardware
  • Continue using software that only runs on outdated OS
  • No need to maintain old physical hardware

Testing & Education

  • Safely test software configurations without affecting the host
  • Educators can provide identical pre-configured environments to students
  • QA teams test across many OS and browser combinations

8.3 Limitations of Virtual Machines

  • Performance overhead: The hypervisor adds a software layer between the VM and hardware — VMs are slower than running directly on hardware, especially for CPU-intensive tasks.
  • Resource contention: Multiple VMs share the physical CPU, RAM, and storage — if one VM uses many resources it can starve others.
  • Hardware access: VMs cannot directly access all hardware features — GPU-intensive tasks (gaming, 3D rendering) perform poorly in VMs.
  • Complexity: Managing many VMs and the hypervisor itself requires specialist knowledge.
  • Security: If the hypervisor itself is compromised, all VMs running on it are potentially compromised (VM escape attacks).

8.4 The Java Virtual Machine (JVM)

The JVM is a special type of virtual machine. Java source code is compiled to platform-independent bytecode (not machine code for any specific CPU). The JVM interprets and executes this bytecode on whatever hardware it is installed on.

The key benefit is "Write Once, Run Anywhere": compile your Java code once and the same bytecode runs on any platform that has a JVM installed, without recompilation.

✏ Practice Questions

(a) Define a virtual machine and explain the role of a hypervisor.
(b) State two benefits and one limitation of using virtual machines in a data centre.
(c) Describe how the Java Virtual Machine allows Java programs to run on different operating systems.

(a) A virtual machine is a software emulation of a complete computer system — it behaves like a real computer but runs within software on a physical host machine [1]. A hypervisor is software that creates and manages VMs, allocating physical hardware resources (CPU, RAM, storage) between them [1].

(b) Benefits (any two): (1) Server consolidation — many virtual servers run on fewer physical machines, reducing hardware and energy costs. (2) Isolation — a failure or security breach in one VM does not affect others. (3) Snapshots — VMs can be saved and restored quickly. (4) Easy migration — VMs can be moved between physical hosts with minimal downtime. [1 each]
Limitation (any one): (1) Performance overhead — the hypervisor adds latency; VMs are slower than native hardware. (2) Resource contention — VMs compete for physical hardware. (3) Complexity of management. [1]

(c) Java source code is compiled to platform-independent bytecode [1]. The JVM is available on many different operating systems (Windows, Linux, macOS, Android) [1]. When a Java program runs, the JVM on that platform interprets/executes the same bytecode, translating it into the appropriate native machine code for the underlying hardware [1] — so the same bytecode runs on any system without recompilation.

Key Formula

Max memory =
2n bytes
(n = address bus bits)

Memory Speeds

● Registers~1 cycle
● L1 Cache~4 cycles
● L2 Cache~12 cycles
● L3 Cache~40 cycles
● RAM~100 cycles
● SSD~100,000 cycles
● HDD~10,000,000 cycles

Common Mistakes

  • Saying ROM is "faster" than RAM — they are similar; the key difference is volatile vs non-volatile
  • Confusing address bus (sends location) with data bus (sends content)
  • Saying more cores always = faster — only for multi-threaded programs
  • Saying virtual memory is "fast" — it uses the disk, which is much slower than RAM
  • Confusing Type 1 (bare metal, no host OS) with Type 2 (runs on top of host OS) hypervisors
  • Forgetting the JVM compiles to bytecode first — it is not directly interpreted from source code
