1 Operating System Functions

An operating system (OS) is system software that manages hardware resources and provides services to application programs and users. Without it, each application would need to control the hardware directly — impossible to manage at scale.

2 Memory Management

The OS must share main memory (RAM) among many running processes simultaneously, while keeping each process's data isolated and safe from others.

2.1 Paging

Paging divides both logical memory (the process's view) and physical memory (RAM) into fixed-size blocks:

  • Page — a fixed-size block of the process's logical address space (e.g. 4 KB)
  • Frame — a fixed-size block of physical RAM, the same size as a page
  • The OS maintains a page table per process mapping page numbers to frame numbers
  • Pages do not need to be contiguous in RAM — the page table handles the mapping
  • Eliminates external fragmentation (wasted gaps between allocations)
  • May cause internal fragmentation — last page of a process may not be fully used

2.2 Segmentation

Segmentation divides a process into variable-size logical segments that match the program's structure (e.g. code segment, data segment, stack segment):

  • Each segment has a base address and a length stored in a segment table
  • More natural to the programmer — segments match logical divisions of code
  • Can cause external fragmentation as free gaps accumulate between segments
  • Modern systems often combine paging and segmentation

2.3 Virtual Memory

Virtual memory extends the apparent size of RAM by using secondary storage (hard disk or SSD) as overflow:

  • Pages that are not currently needed are swapped out to a swap file/partition on disk
  • When a process needs a swapped-out page, a page fault occurs — the OS loads it back into RAM
  • Allows running programs larger than physical RAM
  • Disadvantage: disk access is orders of magnitude slower than RAM — too much paging (thrashing) can cripple performance
Thrashing: When the OS spends more time swapping pages in and out than actually executing processes. Occurs when there is far too little RAM for the workload. Adding more RAM or reducing the number of running processes resolves it.

3 Process Management

A process is a program in execution — it includes the program code, current values of registers, stack, and heap. The OS must share the CPU fairly among all processes.

3.1 Process States

At any moment a process is in one of three states:

READY
Waiting for CPU
scheduled →→
RUNNING
On the CPU
I/O wait →→
BLOCKED
Waiting for I/O
  • Ready → Running: the scheduler picks this process and gives it CPU time
  • Running → Blocked: process requests I/O (e.g. disk read) and must wait
  • Running → Ready: time slice expires (preemption) — another process gets the CPU
  • Blocked → Ready: I/O completes — process can run again when the CPU is free

3.2 Scheduling Algorithms

AlgorithmHow it worksAdvantageDisadvantage
Round Robin Each process gets a fixed time slice (quantum) in turn. When the quantum expires, the next process runs. Fair — every process gets CPU time. Good response for interactive systems. High context-switch overhead if quantum is very short. Long processes take many turns.
Shortest Job First (SJF) The process with the shortest estimated run time is selected next. Minimises average waiting time — theoretically optimal. Requires knowing job length in advance. Long processes may starve (never get CPU).
Priority Scheduling Each process has a priority number. Highest priority runs first. Equal priorities use round-robin. Critical OS processes can be guaranteed fast response. Low-priority processes may starve. Priorities must be set carefully.
Context switch: When the OS switches the CPU from one process to another, it saves the current process's register values (PC, ACC, SR, etc.) to its Process Control Block (PCB) and loads the next process's saved state. This overhead is why very short time slices are inefficient.

3.3 Multi-programming vs Multi-tasking

  • Multi-programming: keeping several processes in memory so the CPU is always busy — when one process waits for I/O, another runs
  • Multi-tasking: rapidly switching the CPU between processes to give the illusion of simultaneous execution (on a single-core CPU)
  • Multi-processing: true parallel execution on multiple CPU cores

4 File & Device Management

4.1 File Management

The OS provides a file system — a structured way to store, name, organise, and retrieve data on secondary storage.

  • Hierarchical directory structure — folders (directories) contain files and sub-folders, forming a tree from a root
  • File permissions — read (r), write (w), execute (x) per user/group/other
  • File Allocation Table (FAT) — records which disk clusters belong to each file
  • Access methods:
    • Sequential access — records read/written in order (e.g. tape, log files)
    • Random (direct) access — jump to any record directly by address (e.g. database)
  • File metadata — filename, size, timestamps, permissions, owner, type

4.2 Device Management

The OS hides hardware complexity from applications through device drivers:

  • Device driver — software that translates generic OS commands into device-specific signals. A new peripheral only needs a driver, not a rewrite of all applications.
  • I/O spooling — jobs (e.g. print jobs) are queued in a spool buffer so the application can continue without waiting for the slow device. The OS feeds the spool to the device as it becomes ready.
  • Plug and Play — the OS automatically detects new hardware, selects, and installs the correct driver without user intervention.
  • Interrupt-driven I/O — a device raises an interrupt when it is ready rather than the CPU constantly polling the device (busy-wait), freeing the CPU for other work.
Exam point: The key reason for device drivers is hardware abstraction — applications use one standard interface regardless of the actual device model. Without drivers, every application would have to handle every possible hardware variant.

5 Language Translators

A language translator converts source code written by a programmer into a form the computer can execute. There are three types.

TranslatorInputOutputKey characteristic
Assembler Assembly language (mnemonics) Machine code One-to-one translation: each mnemonic → one machine code instruction. Very fast; no optimisation.
Compiler High-level source code Machine code executable file Translates the entire program before execution. Produces a standalone executable. Fast at runtime; errors reported after full analysis.
Interpreter High-level source code Immediate execution (no file) Translates and executes one statement at a time. Slower at runtime; reports errors as soon as the line is reached. Easier to debug interactively.

5.1 Assembler

The assembler performs a two-pass process:

  • Pass 1: scan the source to build a symbol table — records all label names and their addresses
  • Pass 2: translate each mnemonic into its binary opcode, replacing labels with addresses from the symbol table

Assembly language is machine-specific — code written for one CPU architecture will not run on another without rewriting.

6 Stages of Compilation

A compiler translates the entire source program through a sequence of stages. Each stage transforms a representation of the program and passes it to the next.

1. Lexical Analysis
The source text is scanned and split into tokens (keywords, identifiers, operators, literals). Whitespace and comments are removed. A symbol table is started to record identifiers. Lexical errors (invalid characters) are reported here.
2. Syntax Analysis
Tokens are checked against the grammar rules of the language using a parser. A parse tree (or Abstract Syntax Tree) is built representing the program structure. Syntax errors (e.g. missing semicolon, mismatched brackets) are reported here.
3. Semantic Analysis
The parse tree is checked for meaning — type checking, variable declarations, scope rules, function signatures. Detects semantic errors such as adding a string to an integer, or using an undeclared variable.
4. Intermediate Code Generation
The validated parse tree is converted to a platform-independent intermediate representation (e.g. three-address code or bytecode). This decouples the front end (language-specific) from the back end (architecture-specific).
5. Code Optimisation
The intermediate code is transformed to run faster or use less memory — without changing what the program does. Examples: removing dead code, constant folding (3+4 → 7), loop unrolling, inlining small functions.
6. Code Generation
The optimised intermediate code is translated into target machine code for the specific CPU architecture. Register allocation decides which values live in registers vs memory. The result is an executable object file.
Exam tip: The most commonly tested stages are Lexical Analysis, Syntax Analysis, and Semantic Analysis. Know what each stage checks for and what type of error it produces. You may be asked to state at which stage a given error would be detected.

6.1 Symbol Table

The symbol table is built during lexical analysis and used throughout compilation. It records:

  • Identifier names (variable names, function names)
  • Data type (integer, string, boolean…)
  • Scope (global, local to a function)
  • Memory address or register assignment

During semantic analysis, the compiler looks up the symbol table to check types match, variables are declared before use, and functions are called with the correct number and types of arguments.

Error classification worked example

Consider: DECLARE x : INTEGER   x ← "hello"

✗
Not a lexical error — all characters are valid tokens (keywords, identifiers, string literal).
✗
Not a syntax error — the statement x ← "hello" is grammatically valid assignment syntax.
✓
Semantic error — assigning a STRING to an INTEGER variable violates type rules. Detected during semantic analysis when the type of the right-hand side is checked against the declared type of x.

7 Compiler vs Interpreter

FeatureCompilerInterpreter
Translation time Translates entire program once before running Translates one statement at a time during execution
Output Standalone executable machine code file No persistent output — executes directly
Execution speed Fast at runtime — machine code runs directly on CPU Slower — translation overhead on every run
Error reporting All errors reported after full analysis; harder to locate Stops at first error and reports its line immediately
Debugging Harder — must recompile after each fix Easier — run partially, inspect variables interactively
Distribution Distribute executable — source code hidden Must distribute source code (or bytecode + runtime)
Portability Executable is platform-specific; recompile for each target Source runs on any platform with the interpreter installed
Memory use Only executable in memory at runtime Interpreter and source must both be in memory
Examples C, C++, Rust, Pascal Python, Ruby, JavaScript (traditionally)
Modern blurred line: Many modern languages use a hybrid approach — Java and Python compile to bytecode first, then interpret (or JIT-compile) that bytecode at runtime. The Cambridge exam treats compiler and interpreter as distinct approaches, so use the table above.

8 Utility Software

Utility programs are system software tools that help maintain, configure, and optimise the computer. They sit between the OS and application software.

UtilityPurpose
Disk Defragmenter Rearranges fragmented files on an HDD so each file's blocks are contiguous, reducing seek time. Not useful or applicable to SSDs.
Disk Formatter Creates a file system structure on a storage device (e.g. FAT32, NTFS, ext4), making it ready to store files.
Backup Software Copies files to a separate location to protect against data loss. Types: full backup, incremental (only changes since last backup), differential.
Anti-malware / Antivirus Detects, quarantines, and removes malicious software. Uses signature databases and heuristic analysis to identify threats.
File Compression Reduces file size using lossless algorithms (e.g. ZIP/DEFLATE) to save disk space and speed up transfers.
Encryption Tool Encrypts files or entire drives (e.g. BitLocker, VeraCrypt) to protect data if the device is lost or stolen.
System Monitor Displays real-time information about CPU usage, memory use, running processes, and network activity. Helps diagnose bottlenecks.
Debugger Allows a developer to run a program step-by-step, set breakpoints, inspect register and variable values, and trace errors.
Exam tip: Be ready to describe the purpose of a utility — not just name it. "A disk defragmenter rearranges file fragments so data is stored contiguously, reducing the time the read/write head needs to move" scores more than "it tidies files."

Question 1 · 4 marks

State four functions of an operating system. [4]

Award 1 mark each for four of:

  • Memory management — allocates RAM to processes, implements paging/virtual memory
  • Process management — schedules CPU time, handles multi-tasking
  • File management — maintains a directory structure, controls file permissions
  • Device management — provides device drivers as a hardware interface
  • Security — user authentication and access control
  • Provides a user interface (GUI or CLI)
  • Handles interrupts
Question 2 · 6 marks

(a) Describe what is meant by paging in memory management. [3]

(b) State one advantage and one disadvantage of paging compared with segmentation. [2]

(c) Explain what happens when a page fault occurs. [1]

(a) Award up to 3:

  • Physical memory (RAM) is divided into fixed-size blocks called frames [1]
  • The process's logical memory is divided into equal-size blocks called pages [1]
  • The OS maintains a page table mapping page numbers to frame numbers; pages need not be contiguous in RAM [1]

(b)

  • Advantage: no external fragmentation — fixed-size pages fit exactly into fixed-size frames [1]
  • Disadvantage: internal fragmentation — the last page of a process may not be completely filled, wasting some memory [1]

(c) The required page is not currently in RAM; the OS must load it from the swap file on disk into a free frame before the process can continue. [1]

Question 3 · 6 marks

Describe the stages of compilation in order, stating the type of error detected at each relevant stage. [6]

Award 1 mark each for six of the following points:

  • Lexical analysis: source code tokenised; whitespace and comments removed; symbol table started [1]
  • Detects lexical errors (invalid characters / unrecognised tokens) [1]
  • Syntax analysis: tokens checked against grammar; parse tree / AST constructed [1]
  • Detects syntax errors (grammatically incorrect statements) [1]
  • Semantic analysis: parse tree checked for meaning — type checking, undeclared variables, scope rules [1]
  • Detects semantic errors (type mismatches, undeclared variables) [1]
  • Intermediate code generation: platform-independent representation produced [1]
  • Code optimisation: intermediate code improved — dead code removal, constant folding, loop optimisation [1]
  • Code generation: target machine code produced for the specific CPU [1]

Accept any 6 correct points.

Question 4 · 4 marks

State two advantages of a compiler over an interpreter, and two advantages of an interpreter over a compiler. [4]

Compiler advantages (any 2):

  • Compiled program executes faster — runs directly as machine code with no translation overhead at runtime
  • Source code does not need to be distributed — only the compiled executable is shared, protecting intellectual property
  • Only needs to be compiled once; subsequent runs need no translator

Interpreter advantages (any 2):

  • Easier to debug — stops and reports an error immediately at the offending line
  • More portable — source code runs on any machine that has the interpreter installed
  • Interactive development — code can be tested and changed without a full compilation step
Question 5 · 3 marks

Describe the purpose of three different types of utility software. [3]

Award 1 mark each for a correct description (not just a name) of any 3 of:

  • Disk defragmenter: rearranges file fragments on an HDD so data is stored contiguously, reducing read/write head movement and access times
  • Backup software: copies files to a separate location to protect against data loss from hardware failure or corruption
  • Anti-malware: scans for, detects, and removes malicious software using signature databases and heuristic analysis
  • File compression: reduces file size using lossless algorithms to save storage space and speed up transfers
  • Encryption tool: encrypts files or drives so data cannot be read without the correct key, protecting against unauthorised access
  • System monitor: displays real-time CPU, memory, and process usage to help diagnose performance bottlenecks

Common Mistakes

  • Saying virtual memory is fast — it uses disk, which is much slower than RAM
  • Confusing paging (fixed-size) with segmentation (variable-size)
  • Saying the interpreter produces an executable — it doesn't
  • Placing type-checking in syntax analysis — it belongs to semantic analysis
  • Saying a compiler is "better" — compilers and interpreters each have advantages in different contexts
